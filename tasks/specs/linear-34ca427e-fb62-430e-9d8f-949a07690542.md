## Evaluation Decision
- Adopt: advisory diagnostics only. Imported sessions can help reconstruct external-agent-originated context because they preserve source path, cwd, title, user/assistant message text, timestamps, and an explicit import marker.
- Defer: any CO automation or helper that imports fixture sessions into task artifacts. This needs a later canary with temp `CODEX_HOME` and a proof of net-new value over existing manifests.
- Reject: provider-worker resume authority, provider-intake authority, hook continuation authority, Linear transition authority, and broad historical migration.
- Follow-up: no implementation follow-up was created because the completed evaluation did not prove a concrete CO automation gap.

- `rust-v0.128.0` release notes list external agent session import, background imports, and imported-session title handling.
- Current local `codex-cli 0.130.0` still exposes `external_migration` as experimental/default-off.
- Current generated app-server TypeScript schema exposes `externalAgentConfig/detect`, `externalAgentConfig/import`, `externalAgentConfig/import/completed`, migration item type `SESSIONS`, and session fields `path`, `cwd`, `title`.
- Upstream `rust-v0.128.0` source/test evidence shows recent `.claude/projects/**.jsonl` detection, 50-session/30-day candidate limits, existing-cwd filtering, duplicate suppression by source hash ledger, custom-title/ai-title/first-user title precedence, rollout item conversion, `<EXTERNAL SESSION IMPORTED>` marker insertion, and thread read/resume after import.
- Boundary: imported sessions preserve enough provenance for diagnostics, but not CO run manifest lineage, child-lane lineage, PR feedback status, provider-intake truth, hook continuation state, or Linear workpad authority.
