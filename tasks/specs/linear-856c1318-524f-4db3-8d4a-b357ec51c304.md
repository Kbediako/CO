# Technical Specification

## Context

The implementation-docs archive policy already includes `docs/PRD-*`, `docs/TECH_SPEC-*`, and `docs/ACTION_PLAN-*`, but the current archiver fails to archive linked docs matching those patterns. The issue evidence and current script behavior point to plain `*` glob handling in the regex conversion path, not to an intentional policy narrowing.

## Requirements

1. Preserve the current broader archive scope already declared in `docs/implementation-docs-archive-policy.json`.
2. Ensure policy patterns using plain `*` match linked implementation docs consistently during task-linked doc extraction.
3. Ensure indexed task doc candidates are canonicalized to repo-relative paths and skipped when they resolve outside the repo root.
4. Ensure canonical `tasks/index.json` path fields (`paths.docs`, `paths.task`, `paths.spec`, `paths.agent_task`) participate in task-linked doc discovery alongside legacy fields.
5. Preserve the current archive payload-before-stub behavior.
6. Add focused regression coverage that proves linked PRD/TECH_SPEC/ACTION_PLAN docs archive into payloads and stubs when eligible, and that explicit task packet paths do not fall into stray archiving.
7. Keep the issue in an active state until a PR is attached and the team review handoff is ready.
