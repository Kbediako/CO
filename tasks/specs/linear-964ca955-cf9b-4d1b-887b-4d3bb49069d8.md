# Technical Specification

## Context

Symphony’s Elixir `before_remove` hook closes open PRs for the current branch before the workspace is removed and treats failures as non-fatal. CO already has the terminal release and workspace cleanup path for provider-owned Linear issues, but that path currently deletes the worktree directly. `CO-5` closes the remaining parity gap by adding a metadata-driven terminal cleanup hook that runs before workspace deletion, closes only matching attached open GitHub PRs, and surfaces failures without crashing the control host.

## Requirements

1. Preserve cleanup ownership in the terminal provider release seam: `providerIssueHandoff` decides when terminal cleanup runs and `workspacePath` remains the bounded provider-worktree deletion helper.
2. Extend the effective `provider-linear-worker` config to carry terminal cleanup metadata, and load it from the last known good snapshot.
3. Run terminal cleanup before `cleanupProviderWorkspace()` removes the provider worktree.
4. Resolve the current workspace branch from the worktree itself and use it to match attached GitHub PR URLs from the current Linear issue.
5. Close only attached GitHub PRs whose head branch matches the workspace branch and whose state is still open.
6. Use a concise machine-generated close reason aligned with the Symphony baseline.
7. Treat cleanup inspection/close failures as non-fatal: log them, surface them in machine-readable control-host state, and continue release cleanup.
8. Cover no attached PR, already-closed PR, successful close, and close-hook failure paths with focused tests.

## Current Truth

- `orchestrator/src/cli/control/providerIssueHandoff.ts` already decides when a released claim should clean up its workspace.
- `orchestrator/src/cli/run/workspacePath.ts` currently removes the worktree with `git worktree remove`, `rm`, and `worktree prune`, but has no pre-remove hook.
- `orchestrator/src/cli/control/providerLinearWorkflowFacade.ts` already resolves Linear issue context including attachments and canonical GitHub PR URLs.
- `orchestrator/src/cli/control/providerWorkflowConfigStore.ts` already owns the effective `provider-linear-worker` config snapshot and the control-host `provider_workflow` payload.
- `codex.orchestrator.json` currently defines `provider-linear-worker` but exposes no terminal cleanup metadata.
