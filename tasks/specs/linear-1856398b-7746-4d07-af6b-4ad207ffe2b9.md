## Technical Requirements
- Functional requirements:
  1. Add a canonical machine-readable posture matrix or equivalent structured source. Preferred shape is a repo-tracked JSON source such as `docs/codex-posture-matrix.json`; an equivalent `docs/docs-catalog.json` policy extension is acceptable if it keeps one authoritative source.
  2. Matrix rows must distinguish at least:
     - active Codex CLI target
     - current model posture
     - default runtime posture
     - explorer_fast/spark exception
     - audited candidate version
     - release-facing workflow compatibility pin
     - cloud-canary candidate pin
     - pack-smoke required capabilities
     - historical/archive evidence status
  3. Docs-hygiene must validate README, docs/README or docs/book/current navigation, public posture, downstream setup docs, version policy, workflow pins, and pack-smoke expectations against the matrix.
  4. Historical release evidence pages must be allowed to remain only when marked historical/archive/demoted in machine-readable metadata or removed from active current-facing navigation.
  5. Workflow pin validation must allow intentional compatibility pins when the matrix records the pin role, version, reason, and required capability.
  6. Pack-smoke validation must keep supported marketplace add behavior (`codex plugin marketplace add`, with legacy `codex marketplace add` fallback where supported) as a fail-closed prerequisite unless an explicit local-dev skip reason is present.
  7. Focused tests must include a stale active `0.124` release evidence/book-navigation fixture that fails without historical/archive status and passes after demotion/status marking.
- Non-functional requirements:
  - deterministic local docs-hygiene behavior without network calls
  - precise error messages naming the drifting surface and expected matrix row
  - no deletion-only remediation path for historical evidence
  - no broad docs freshness sweep as part of this implementation
- Interfaces / contracts:
  - `tasks/index.json` remains canonical under `items[]`
  - docs-hygiene remains the blocking `npm run docs:check` surface
  - docs freshness remains a separate freshness/audience/status signal and should not mask posture disagreement

  - Update `scripts/docs-hygiene.ts` to build one posture contract object and apply it to markdown docs, docs/navigation metadata, workflow YAML, and pack-smoke expectation checks.

    - `version`
    - `model`

- External dependencies / integrations:

  - Parent may reuse existing YAML parsing in tests where available.
