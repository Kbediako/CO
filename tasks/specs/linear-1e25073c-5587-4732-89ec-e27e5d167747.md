## Summary
- Objective: recognize common clean review wording as semantic clean verdicts while preserving fail-closed `unknown` behavior and findings precedence.
- Scope:
  - semantic review verdict parser.
  - review telemetry fixture/test coverage for CO-492 observed clean phrases.
  - negative/finding cases that prove clean wording does not mask P-prioritized or structured findings.
  - provider-worker handoff behavior only insofar as it consumes the existing semantic verdict.
- Constraints:
  - no PR/Linear lifecycle authority changes.
  - no broad review wrapper rewrite.
  - no waiver broadening.
  - no fallback that treats generic `unknown` as clean.

## Requirements
- Add clean-phrase recognition for wording equivalent to:
  - `No actionable defects were found...`
  - `No concrete correctness regressions were found...`
- Keep findings precedence: any structured finding, P-prioritized finding line, or explicit actionable defect signal must produce a findings verdict even when clean-looking prose is also present.
- Preserve true-unknown behavior when the review text lacks a recognized clean signal and lacks findings.
- Keep `review_outcome: clean-success` insufficient by itself when findings are parsed.
- Keep provider-worker review gates requiring semantic clean, findings-free output, or an explicit waiver.

## Validation Plan
- Focused parser tests for the two CO-492 observed phrases.
- Negative tests where clean wording plus a P2/P1/P0 or structured finding still returns findings.
- True-unknown test where unrecognized non-finding text remains `unknown`.
- Existing provider-worker review gate tests should remain green.
- Run the normal provider-worker validation floor before handoff.
