## Summary
- Objective: recognize common clean review wording as semantic clean verdicts while preserving fail-closed `unknown` behavior and findings precedence.
- Scope:
  - semantic review verdict parser.
  - review telemetry fixture/test coverage for CO-492 observed clean phrases.
  - current-head PR review edge cases around labeled actionable defect summaries, validation-only notes, comma-separated defect labels, and benign follow-up sentences.
  - negative/finding cases that prove clean wording does not mask P-prioritized or structured findings.
  - provider-worker handoff behavior only insofar as it consumes the existing semantic verdict.
- Constraints:
  - no PR/Linear lifecycle authority changes.
  - no broad review wrapper rewrite.
  - no waiver broadening.
  - no fallback that treats generic `unknown` as clean.

## Requirements
- Add clean-phrase recognition for wording equivalent to:
  - `No actionable defects were found...`
  - `No concrete correctness regressions were found...`
  - `Read-only inspection... did not identify actionable regressions...`
- Keep findings precedence: any structured finding, P-prioritized finding line, or explicit actionable defect signal must produce a findings verdict even when clean-looking prose is also present.
- Treat comma-separated `actionable defect(s):` labels as explicit defect signals, not clean sentence suffixes.
- Treat inline no-op `actionable defect(s):` labels as clean only when the pre-label prefix is already clean, no-op, validation-only, or benign; unsafe same-line prefixes remain findings.
- Treat numbered list markers before direct `Actionable defects:` labels and clean/no-op prefixes as list syntax, not unsafe inline prefix prose.
- Treat labeled no-op `Actionable defects:` summaries as clean only when the summary is explicitly no-op, including dotted-path scope qualifiers before validation-only notes such as `tests not run` or `I did not run tests`, benign positive follow-up prose, and allowed no-op/benign prefaces before a terminal clean verdict.
- Treat labeled no-op summaries as allowed clean prefaces when followed by a terminal clean verdict sentence.
- Permit a clean first sentence followed by a whitelisted benign positive review sentence only when that benign sentence is the end of the candidate; split clean sentences only count when terminal, preserving `unknown` for unsafe period-separated prose.
- Constrain prefixed clean subjects and terminal split candidates so unsafe semicolon or period-separated lead-ins do not become clean.
- Accept bounded review-subject clean wording, including `Review`, `Reviewer`, `The reviewer`, `Codex`, and bounded/read-only retry subjects.
- Ignore known top-level runtime cleanup noise lines before rejecting natural-language clean verdicts, matching the existing structured JSON noise handling.
- Do not use question marks as clean sentence split boundaries.
- Treat exception wording such as `with one exception` as fail-closed, while keeping exception-named path scopes clean.
- Allow narrow neutral review preambles or headings before a clean verdict without allowing unsafe prose before or after the clean signal.
- Keep terminal clean review-subject sentence splitting aligned with the accepted direct clean-subject grammar, including bare standalone, bounded, and read-only review subjects.
- Preserve question-mark prefixes as non-clean before validation-note handling.
- Preserve true-unknown behavior when the review text lacks a recognized clean signal and lacks findings.
- Keep `review_outcome: clean-success` insufficient by itself when findings are parsed.
- Keep provider-worker review gates requiring semantic clean, findings-free output, or an explicit waiver.

## Validation Plan
- Focused parser tests for the two CO-492 observed phrases.
- Negative tests where clean wording plus a P2/P1/P0 or structured finding still returns findings.
- Regression tests for current-head PR review feedback: comma-separated defect labels, inline no-op label prefixes, numbered clean/no-op list labels, period-separated validation notes, `tests not run` shorthand, dotted-path prefixes before validation/benign trailing clauses, benign positive follow-up prose, benign labeled no-op follow-ups, combined no-op/benign labeled prefaces and benign/validation-only middle notes before terminal clean verdicts, review-subject terminal clean sentences after no-op summaries, bare standalone/bounded/read-only review terminal clean subjects, article-prefixed reviewer wording, runtime cleanup noise near clean verdicts, exception wording, exception-named path scopes, narrow neutral review preambles/headings, unsafe defect-prose preamble negatives, prefixed did-not-identify wording, trailing unsafe sentences, terminal split candidates, unsafe non-terminal split suffixes, unsafe same-line semicolon/period lead-ins, and question-mark prefixes.
- True-unknown test where unrecognized non-finding text remains `unknown`.
- Existing provider-worker review gate tests should remain green.
- Run the normal provider-worker validation floor before handoff.
