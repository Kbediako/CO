## Current Truth
- `acquireLockWithRetry(...)` writes a random owner token to the lock path opened with `wx+`, then returns an `AcquiredLock` wrapper.
- `createLockKeepalive(...)` starts a timer only when `staleMs > 0`; the interval is `Math.max(10, Math.floor(staleMs / 2))`.
- `touchLockHandle(...)` refreshes the file handle mtime and intentionally ignores keepalive touch failures.
- `release()` stops the keepalive timer, closes the handle, and removes the lock only if the file still contains the same owner token.
- `orchestrator/tests/LockFile.test.ts` currently verifies stale orphan recovery and live lock freshness with small millisecond timing windows.
