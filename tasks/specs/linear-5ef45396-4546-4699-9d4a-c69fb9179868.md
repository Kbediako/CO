## Parent Implementation Decision
- `large refactor`: not triggered. The implementation does not add another runtime routing seam; it annotates existing fallback outcomes and updates schema/reporting/tests around the existing `RuntimeFallbackMetadata` payload.
- `minor seam`: acceptable. The change is scoped to one governed surface (`runtime routing`), one lifecycle payload (`runtime_fallback`), and carries CO-396 expiry metadata for retained temporary auto reroutes.
- `remove fallback`: not selected for this pass because the appserver preflight and implicit cloud-default bridges still have live compatibility triggers.
- `expire fallback`: selected for appserver preflight auto reroute and implicit cloud default appserver reroute. Both emit `runtime_fallback.expiry` with owner `CO-396`, review date `2026-05-10`, maximum lifetime `2026-05-26`, removal condition, and validation.
- `justify retaining fallback`: selected for strict blocked cloud/appserver metadata, runtime fallback policy aliases, `RuntimeFallbackMetadata`, and runtime canary scenarios.
