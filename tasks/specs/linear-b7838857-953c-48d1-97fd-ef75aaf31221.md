## Problem
- Ephemeral child-lane workspaces under `.child-lanes/<stream>-<run>` can leave durable entries in `~/.codex/config.toml` even though stable ancestors such as `/Users/kbediako/Code/CO` are already trusted.

## Contract
- Translation: keep same-issue child lanes and parent-owned acceptance, but stop ephemeral lane paths from polluting global trust/config state.
- Protected terms: `trusted-project pollution`, `child-lane paths`, `global ~/.codex/config.toml growth`, `do not weaken trust boundaries`
- Protected surfaces: `.child-lanes/<stream>-<run>`, `lane_workspace_path`, `CODEX_HOME`, `~/.codex/config.toml`, `resolveCodexHome`, `provider-linear-child-lane-proof.json`
- Reject: disabling trust checks, trusting all `.child-lanes/*`, collapsing lanes into the parent workspace, or broadening into generic global config redesign.
- Non-goals: auth/profile work, unrelated provider workflow changes, generic Codex defaults cleanup.
- Parity: current child lanes clone into `.child-lanes/<stream>-<run>`, inherit `CODEX_HOME`, and can append unique trusted project entries; reference is that a trusted stable ancestor is sufficient and unrelated roots remain untrusted; target is to remove only the current redundant child-lane trust residue while preserving fail-closed trust boundaries.
- Not done if: each new lane can still add durable trust residue, only manual cleanup exists, or the fix widens trust.

## Requirements
- Remove only the current redundant child-lane trust entry when a separate trusted ancestor exists.
- Preserve unrelated project entries and TOML sections, and persist a failed `provider-linear-child-lane-proof.json` when post-exec artifact export fails.
- Cover exact-lane cleanup, array-table boundaries, quoted `]` headers, nested project subtables, and multiline header-like content in regression tests.

## Validation
- Focused runner validation during implementation: build, targeted lint, targeted vitest, diff budget. Full handoff validation: manifest-backed review, elegance pass, and clean `pr ready-review` drain.
