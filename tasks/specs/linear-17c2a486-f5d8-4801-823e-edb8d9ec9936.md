# Technical Specification

## Context

`CO-55` required screenshot proof, but the resulting proof was still expressed as local artifact paths or external references because CO's current Linear mutation surface only supports text workpads and PR or URL attachments. Linear itself already supports file uploads plus embedded image markdown in issue comments and descriptions. The current gap is therefore not whether Linear can show the screenshot; it is that the CO helper surface cannot yet take a local screenshot artifact and turn it into a visible in-Linear proof entry.

## Requirements

1. The worker-owned Linear helper surface must accept a local screenshot artifact intended for direct Linear embedding.
2. The helper must request a Linear `fileUpload` payload, including signed upload URL, required upload headers, and the resulting `assetUrl`.
3. The worker must upload the screenshot bytes server-side before the workpad comment mutation runs.
4. The final stored workpad comment must reference the resulting Linear `assetUrl` with markdown image syntax so the screenshot renders directly in Linear.
5. The proof contract must be able to state `embedded directly in Linear, not only linked` when the issue requires that stronger proof mode.
6. The mutation result or audit path must preserve enough traceability to tie the updated workpad comment to the uploaded proof asset.
7. Existing PR attachment and external proof URL flows must remain unchanged.

## Planned Interaction Contract

- Workpad-authoring path:
  - worker prepares the workpad body as usual
  - workpad body may contain explicit local screenshot references meant for Linear embedding
  - the helper resolves those local screenshot references before the comment mutation:
    - read local file
    - request `fileUpload`
    - upload bytes via signed `PUT`
    - rewrite markdown image reference to `assetUrl`
- Reviewer outcome:
  - reviewer opens the issue
  - reviewer sees the screenshot directly in the active `## Codex Workpad` comment
  - audit or helper output can identify the uploaded asset URL plus the mutated workpad comment id

## Pre-change Baseline

- `providerLinearWorkflowFacade.ts` currently:
  - reads or updates the single workpad comment
  - validates the workpad section contract
  - supports PR attachment and URL attachment flows
  - does not upload local files or rewrite local screenshot references
- `linearCliShell.ts` currently:
  - routes `issue-context`, `upsert-workpad`, `delete-workpad`, `transition`, `attach-pr`, `create-follow-up`, `runtime-proof`, `child-stream`, and `child-lane`
  - does not expose any local-file upload support today
- `providerLinearWorkflowAudit.ts` currently captures comment ids and attachment ids, but not a Linear proof asset URL
- `skills/linear/SKILL.md` and the provider-worker prompt still frame proof around reviewer-visible proof URLs rather than local screenshot upload into Linear itself
