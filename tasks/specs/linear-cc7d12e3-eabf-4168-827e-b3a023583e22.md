# Technical Specification

## Context

CO already has the primitives needed for both prompt-only and forced unattended review. The review wrapper intentionally prints a handoff prompt in non-interactive mode unless `FORCE_CODEX_REVIEW=1` is set. Gate pipelines already opt into the forced path. Advisory review stays prompt-only and non-blocking. Provider-worker closeout currently sits in between: the worker is unattended, but the repo does not yet say clearly whether its final standalone review should actually execute or merely print a handoff prompt.

## Requirements

1. Keep raw non-interactive `codex-orchestrator review` / `npm run review` handoff-only unless `FORCE_CODEX_REVIEW=1` is present.
2. Preserve explicit forced review execution in unattended `docs-review` and `implementation-gate` lanes.
3. Keep unattended advisory review explicit and prompt-only/non-blocking unless a future issue intentionally changes that contract.
4. Make provider-worker closeout explicit about standalone review becoming an autonomous forced stage before review handoff.
5. Align `codex.orchestrator.json`, `AGENTS.md`, `docs/AGENTS.md`, `docs/standalone-review-guide.md`, and `skills/standalone-review/SKILL.md` to the same matrix.
6. Add focused contract coverage proving gate, advisory, and provider-worker unattended review behavior.
7. Do not relabel the current wrapper behavior as a bug if it remains prompt-only by design outside explicit forced lanes.
8. Keep the issue active until a PR is attached and the team review handoff is ready.

## Current Truth

- `scripts/lib/review-non-interactive-handoff.ts` and `scripts/run-review.ts` intentionally suppress real review execution in unattended mode unless `FORCE_CODEX_REVIEW=1` is present.
- `codex.orchestrator.json` already sets `FORCE_CODEX_REVIEW=1` in `docs-review` and `implementation-gate` review stages.
- `codex.orchestrator.json` keeps `docs-relevance-advisory` non-blocking and currently does not force review execution there.
- `orchestrator/src/cli/providerLinearWorkerRunner.ts` tells workers to validate, handle PR feedback, and stop coding at review handoff, but it does not yet tell them whether standalone review is forced or prompt-only before that handoff.
- Symphony's current Elixir workflow makes unattended worker behavior explicit at the prompt layer: one workpad, explicit review handoff states, and stop-coding boundaries.
- This worker run cannot use spawned subagents, so delegation guard must be satisfied with an explicit override rather than fabricated delegation evidence.
