# Technical Specification

## Context

The current diff-budget guard is still useful, but the local working-tree default is no longer truthful on stacked CO lanes. Today `scripts/diff-budget.mjs` resolves `origin/main` as the default base and enforces one hard `25`-file / `800`-line budget against that broad aggregate. That remains a truthful hard surface for explicit PR-base and CI use when `BASE_SHA` is present, but it becomes low-signal on stacked local lanes where the current reviewable slice is smaller than the full branch history. Recent closeout evidence shows both sides of the problem: a focused lane passes cleanly against `origin/main`, while long-lived stacked lanes require overrides with reasons that explicitly say the current signal is not lane-local.

## Requirements

1. Local default working-tree diff-budget runs must hard-gate the current reviewable head scope instead of always hard-gating the full `origin/main` aggregate.
2. Explicit `--commit`, explicit `--base`, and CI-provided `BASE_SHA` / `DIFF_BUDGET_BASE` paths must remain hard-gated against their requested scope.
3. When local auto mode detects a broader stacked aggregate than the current head scope, the output must still report the aggregate as advisory context rather than hiding it.
4. Default thresholds must align with the current standalone-review large-scope policy where possible; the line threshold should no longer lag the documented `1200`-line standard.
5. Operator-facing docs must make the hard local scope versus hard CI/base scope contract explicit.
6. Focused regressions must cover:
   - explicit hard base/commit behavior
   - stacked-lane local auto behavior
   - threshold alignment and output wording
   - unborn-repository staged-content line accounting in both hard diff-budget and uncommitted review scope modes
7. Keep the issue active until a PR is attached and the review handoff is ready.

## Current Truth

- `scripts/diff-budget.mjs` only distinguishes explicit `--commit` runs from a generic working-tree run that defaults to `origin/main` when no explicit base is provided.
- `codex.orchestrator.json` `implementation-gate` runs `node scripts/diff-budget.mjs` with no explicit scope hint, so local implementation-gate inherits the generic working-tree behavior.
- `.github/workflows/core-lane.yml` sets `BASE_SHA` from the pull request base SHA before running the script, so CI already has a truthful explicit-base hard gate.
- `scripts/run-review.ts` also routes through `scripts/diff-budget.mjs` when diff-budget is not already pipeline-owned, so any local default change must keep wrapper behavior coherent.
- Recent baseline evidence is captured in `out/linear-74d145eb-305b-4b27-be84-21c248b22e4d/manual/20260325T070238Z-baseline-audit.md`.
