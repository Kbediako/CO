# Technical Specification

## Context
`CO-173` fixes provider-worker forced standalone reviews where repeated `command-intent` boundaries replace actual review verdicts. The boundary remains correct; the defect is a routine manual fallback.

## Requirements
1. Preserve strict `validation-suite` / `validation-runner` blocking, while letting one stricter retry produce `clean-success` or `bounded-success`.
2. Make bounded no-validation guidance visible or enforceable for `--uncommitted`, `--base`, and `--commit`.
3. Keep `failed-boundary`, `failed-other`, `bounded-success`, and `clean-success` telemetry truthful, with command-intent reason counts separate from product findings.
4. Cover scoped context transport, retry behavior, telemetry interpretation, and `docs/standalone-review-guide.md`.

## Protected Expectations
- `FORCE_CODEX_REVIEW=1`, `codex-orchestrator review`, and `npm run review` remain mandatory pre-handoff surfaces.
- Protected terms keep their meanings: `failed-boundary`, `command-intent`, `validation-suite`, `validation-runner`, `bounded-success`, `clean-success`, `prompt_delivery=artifact-only`, `reviewer_visible_context_transport=scoped-title`.
- Do not permit validation suites, hide boundary telemetry, normalize manual fallback, or conflate CO-102 freshness debt with this bug.

## Parity / Alignment Matrix
- Current truth: scoped review launches can leave critical no-validation guidance artifact-only, while telemetry shows recurring validation-command `failed-boundary` outcomes.
- Reference truth: provider-worker closeout requires a forced standalone review verdict before `Human Review` / `In Review`.
- Target truth: validation attempts are blocked or retried once into a verdict while preserving boundary evidence.
- Out of scope: broad review taxonomy redesign, validation execution inside bounded review, and CO-92/CO-94 product scope.

## Not Done If
- Routine closeout still needs manual fallback after `npm run test`, `npx vitest`, or equivalent attempts.
- The patch only changes docs.
- Boundary enforcement is hidden, weakened, or bypassed.

- Use same-issue child-lane/docs-review manifests as delegation evidence.
- Run focused review wrapper tests and repro scoped forced-review runs.
- Run full repo gates, forced standalone review, elegance pass, and `pack:smoke` before handoff.
