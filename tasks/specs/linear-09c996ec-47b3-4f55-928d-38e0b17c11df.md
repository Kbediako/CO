# Task Spec - CO-275 marketplace-dependent downstream-smoke policy closeout

## Summary
`CO-275` records the policy decision that `CO-269` deferred: post-`0.121.0` marketplace-dependent downstream smoke should use the validated `Codex CLI 0.122.0` plugin marketplace surface rather than keep the old `codex marketplace add` path as the steady-state requirement.

## Protected Scope
- Preserve: `Codex CLI 0.122.0`, `Codex CLI 0.121.0`, `codex marketplace add`, `codex plugin marketplace add`, `pack:smoke`, `scripts/pack-smoke.mjs`, `tests/pack-smoke.spec.ts`, `.github/workflows/core-lane.yml`, `.github/workflows/release.yml`, `.github/workflows/pack-smoke-backstop.yml`, `@openai/codex@0.121.0`, `@openai/codex@0.122.0`, `CO-196`, `CO-217`, and `CO-269`.
- Reject: active target promotion, cloud-canary policy churn, provider-worker runtime work, package architecture redesign, or optional marketplace smoke.

## Target Truth
- `scripts/pack-smoke.mjs` probes and invokes `codex plugin marketplace add`.
- `tests/pack-smoke.spec.ts` asserts release-facing smoke workflows install `@openai/codex@0.122.0` before `npm run pack:smoke`.
- `.github/workflows/core-lane.yml`, `.github/workflows/release.yml`, and `.github/workflows/pack-smoke-backstop.yml` run marketplace smoke against `@openai/codex@0.122.0`.
- `CO-196` remains the packaged marketplace architecture lineage.
- `CO-217` remains the mandatory marketplace-smoke / explicit non-coverage lineage.
- `CO-269` remains the source of the temporary `0.121.0` hold and follow-up creation.

## Acceptance Criteria
1. Prior `0.121.0` versus `0.122.0` old-command evidence remains cited.
2. The final policy decision is replacement-contract, not undocumented hold.
3. Script, tests, workflows, and docs agree on `codex plugin marketplace add` and `@openai/codex@0.122.0`.
4. `CO-196`, `CO-217`, and `CO-269` remain explicit.

## Validation
- Protected-term scan over the packet and implementation surfaces.
- Focused pack-smoke tests.
- Required repo gates before review handoff.
