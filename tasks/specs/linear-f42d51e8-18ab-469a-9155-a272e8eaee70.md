## Summary
- Objective: update `codex-orchestrator hygiene quota` so an available bounded degraded `co-status --format json` machine-status payload is classified as non-healthy partial status rather than `unavailable`.
- Scope: quota hygiene co-status classification, summary/finding evidence, focused quota-hygiene tests, docs packet, and validation.
- Constraints: preserve fail-closed stronger classifications for stale endpoints, unhealthy live hosts, auth failures, dead endpoints, true unavailable status, active failed runs, and real quota-burning processes.

## Technical Requirements
1. `classifyCoStatusDataset` must keep `healthy` only when `degraded_read` is absent.
2. Known stale endpoint degraded-read reasons remain `stale_endpoint`.
3. Known unhealthy live-host degraded-read reasons remain `unhealthy_live_host`.
4. Known auth/unavailable degraded-read reasons remain `unavailable`.
5. Other bounded degraded-read reasons, including `ui_request_timeout`, become an available non-healthy degraded machine-status classification.
6. `summarizeCoStatus` must include `degraded_read_reason`, `degraded_read_source`, and `degraded_read_freshness_verdict`.
7. The finding summary for available degraded machine status must distinguish it from unavailable status and point to the producer reason/source/freshness.
8. `collectCoStatusLiveTokens` must continue returning `[]` for any degraded read.
9. Quota hygiene remains read-only and zero-model for this path.
