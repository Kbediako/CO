## Requirements
- Default runtime-proof reviewer URL validation must remain deterministic and auditable from checked-in permit data plus explicit blocked-host/IP rules.
- If DNS-aware validation is added, it must be explicit opt-in, worker-local in scope, and fail closed on resolver failure, zero answers, or any non-public answer.
- Split-horizon or mixed public/private answers must not be treated as reviewer-reachability success.
- Docs and focused tests must cover deterministic behavior, DNS-public success, DNS lookup failures, and non-public-resolution failures.
