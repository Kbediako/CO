## Summary
- Objective: prevent duplicate follow-up issues when `create-follow-up` already created a Linear issue but failed later with `linear_follow_up_description_update_incomplete`.
- Scope: create-follow-up retry/reconciliation logic, focused tests, docs/task registry evidence, and PR lifecycle.
- Constraints: preserve fail-closed verification and existing canonical-owner retry behavior.
