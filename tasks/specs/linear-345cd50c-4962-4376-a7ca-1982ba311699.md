# Technical Specification

## Context

CO-445 targets the provider closeout lifecycle after a metadata PR has merged. The CO-444 incident showed that a successful owner-renewal merge can be followed by provider closeout moving the same issue to terminal `Done` while `origin/main` still configures that issue as the `docs:freshness:maintain` `owner_issue`. The next maintenance pass then reports a terminal configured owner via `configured_owner_terminal`, may escalate to `block_unowned_repo_debt`, and can still show `blocking_changed_paths=[]`.

The implementation surface is `providerMergeCloseout.ts`. The closeout code already owns the merge/shared-root/Linear terminal transition flow, so it must check live owner status before attempting `Done`. This lane must not weaken docs freshness policy or hide terminal-owner evidence; it must prevent provider closeout from creating that terminal-owner condition for an issue that is still the live owner.

## Technical Requirements
- Functional requirements:
  1. Resolve `docs:freshness:maintain` owner evidence after merge/shared-root reconciliation and before the `Done` transition in `providerMergeCloseout.ts`.
  2. If policy and maintenance evidence show the current issue is still the live `docs:freshness:maintain` owner, skip `Done`, move or hold the issue in a non-terminal owner-bearing state such as `Blocked`, and return an action-required closeout record.
  3. Preserve machine-readable evidence fields for `policy_owner_issue`, `policy_canonical_owner_key`, `owner_issue`, `owner_issue_action`, `owner_issue_verification`, `freshness_decision`, `candidate_cohorts`, and `blocking_changed_paths`.
  4. If ownership has been re-homed or retired, preserve normal terminal `Done` closeout.
  5. Keep `configured_owner_terminal` and `block_unowned_repo_debt` as visible fail-closed docs freshness outcomes.
  6. Adjust provider issue observability so merged action-required closeout is not projected as completed.
- Non-functional requirements:
  - no provider merge readiness gate weakening outside the terminal owner guard
  - no docs freshness policy or cohort metadata churn
  - smallest patch across provider closeout, observability projection, and focused tests
  - evidence remains usable in workpad/manifest/debug output
- Interfaces / contracts:
  - `providerMergeCloseout.ts` closeout record gains docs freshness owner evidence
  - `docs:freshness:maintain -- --format json` remains the owner evidence command
  - Linear transition target for live-owner hold is the team's `Blocked` state when exposed
  - observability read model treats closeout action-required as stalled/watching merge, not completed
