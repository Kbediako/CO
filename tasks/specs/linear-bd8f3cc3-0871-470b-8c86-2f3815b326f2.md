# Technical Specification

## Context
`CO-74` intentionally put `co-status attach` on `primary scrollback`, while `CO-67` fixed a different paused duplication seam. The current defect is the remaining attach writer strategy: attach is already on primary, but it still appends a full new `CO STATUS` frame each refresh. The same live surface also still carries the readability/truth defects explicitly listed on `CO-97`.

## Requirements
1. `co-status attach` remains read-only for an already-running control-host.
2. Live attach preserves primary scrollback without accumulated duplicate full-frame `CO STATUS` history.
3. The fix does not simply revert attach to alternate screen.
4. Attach live redraw uses a bounded in-place or pinned-region strategy.
5. Launch-mode alternate-screen behavior remains truthful and distinct.
6. Countdown, runtime/age, retry, and rate-limit reset text uses higher-order readable durations where needed.
7. Codex / Linear rate-limit output matches the requested percentage and spacing semantics.
8. Retry/event text is more meaningful, and presenter/read-model widening stays evidence-driven.
9. Focused tests and real-device screenshot proof cover the bounded contract.

## Design
- Keep attach on primary and launch on alternate.
- Rewrite the attach live block in place only when both the previous and next frame fit the viewport; otherwise use a fresh `ANSI_CLEAR_HOME` rewrite.
- Count pinned attach rows by terminal display width, not UTF-16 length.
- Centralize higher-order duration formatting inside the dashboard surface.
- Keep source selection unchanged unless validation proves a separate stale presenter defect remains.
- Capture reviewer-usable proof directly in Linear with explicit cleanup and payload cross-checks.
