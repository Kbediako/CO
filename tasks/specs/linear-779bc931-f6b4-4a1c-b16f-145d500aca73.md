# TECH_SPEC - CO-492 advisory persisted goal evidence capture

## Contract
Provider-worker runs may capture persisted `/goal` state as optional manifest `goal_evidence` and a compact workpad summary. The evidence must always carry `authority=advisory_only`; Linear remains source of truth for workflow lifecycle.

## Protected Terms
- persisted `/goal`
- goals feature
- app-server APIs
- model tools
- provider-worker run evidence
- manifest `goal_evidence`
- workpad summary
- `advisory_only`
- Linear remains source of truth

## Explicit Authority Rejections
Goal state must not authorize:
- Linear transitions
- PR attachment
- review handoff
- ready-review success
- merge closeout
- hook recovery success
- long-poll terminal status
- hook/resume control integration
- TUI automation

## Requirements
1. `goal_evidence` is optional and advisory.
2. Capture source must be recorded as model tools, app-server APIs, or unavailable.
3. Available evidence must include current thread/objective/status/provenance where the source can supply it.
4. Unavailable, disabled, stale, complete, paused, budget-limited, or thread-mismatched evidence must be recorded as advisory context or omitted with an explicit reason; it must not block lifecycle progress.
5. `capture_timestamp` records provider-worker evidence observation time; goal `created_at` and `updated_at` remain separate goal provenance.
6. Workpad summary must be compact and must not become sole proof.
7. Lifecycle decisions continue to depend on Linear/workpad/PR/review/check evidence.
