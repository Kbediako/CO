# Technical Specification

## Context

`CO-58` repaired the evaluation baseline for provider-worker `eval:test` by restoring truthful `NODE_PATH` search roots across copied fixtures, ancestor repo roots, external worktrees, symlinked paths, and submodule boundaries. A standalone review on 2026-04-02 found one remaining same-family gap outside that lane's accepted scope: `.git` file layouts created by `git clone --separate-git-dir <gitdir>` can still resolve a `.git` directory whose parent is not an ancestor of the checkout, and `evaluation/harness/env.ts` currently treats that parent as the repo root. That lets `collectNodeModulePaths()` walk above the checkout root and append unrelated ancestor `node_modules` entries to `NODE_PATH`.

## Requirements

1. Reproduce the current external gitdir checkout leak with a deterministic harness test or equivalent focused fixture.
2. Update `evaluation/harness/env.ts` so non-ancestor external gitdir layouts stay scoped to the current checkout root instead of the external gitdir parent.
3. Preserve the existing nested worktree, symlinked path, and submodule boundary behavior already covered in `evaluation/tests/harness.test.ts`.
4. Keep the lane docs-first and provider-worker workflow evidence complete: task packet, single workpad, child docs-review evidence, and pre-handoff standalone/elegance review for any non-trivial diff.
5. Validate the repaired branch with `MCP_RUNNER_TASK_ID=linear-f7b20174-c1f3-4277-a803-e22e643372f4 npm run eval:test`.
