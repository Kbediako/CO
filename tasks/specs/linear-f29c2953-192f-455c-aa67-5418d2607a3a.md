# Technical Specification

## Context
Full-suite validation for `CO-216` still fails in `orchestrator/tests/Doctor.test.ts` under suite load even after the operator-autopilot fix is stable.

Evidence from April 17, 2026 in this workspace:

- `npm run test` on the settled `CO-216` diff failed with 5 `Doctor.test.ts` timeouts:
  - `reports direct-dist delegation readiness and initialize latency`
  - `keeps top-level doctor status ok when delegation is advisory but the rest of the install is healthy`
  - `keeps overall doctor status at warning when providers are incomplete`
  - `reports provider readiness when the repo is seeded and env is configured`
  - `resolves provider readiness from the repo root when doctor runs in a nested directory`
- The same diff then passed `npx vitest run --config vitest.config.core.ts orchestrator/tests/Doctor.test.ts` in isolation: `47 passed`, duration about `39s`.

This follow-up exists to isolate and fix the suite-load timeout behavior in the doctor coverage so provider-worker validation stops failing for unrelated reasons. The repair is therefore about the mismatch between full-suite and isolated doctor behavior, not about reopening `CO-216` backlog-promotion or manual-demotion policy logic.

## Requirements
1. Reproduce the `Doctor.test.ts` timeout cluster under `npm run test` with concrete evidence from the current suite.
2. Prove whether the root cause is load-sensitive doctor work, environment-sensitive probing, or another suite interaction.
3. Land a bounded fix so `npm run test` no longer times out in the affected doctor cases.
4. Preserve `npx vitest run --config vitest.config.core.ts orchestrator/tests/Doctor.test.ts` passing in isolation after the fix.
5. Keep the follow-up scoped to doctor-suite validation behavior rather than `CO-216` autopilot logic.

## Design
- Investigate the load-sensitive paths exercised by the five named doctor readiness cases before changing code.
- Prefer one bounded stabilization in doctor command logic, shared readiness helpers, or the doctor test harness over broad global timeout increases.
- Preserve doctor signal quality: the fix must not simply delete the failing coverage or declare isolated-file success sufficient.
- Keep the repair local to the doctor suite-load seam so provider-worker validation for unrelated lanes stops failing for unrelated reasons.
