## Fallback Expiry / Refactor Decision
- Applies to fallback, compatibility, legacy, stale, cached, break-glass, or minor-seam behavior? `Yes`.
- Policy guide: `docs/guides/fallback-expiry-and-refactor-policy.md`.
- Required decision: CO-382 creates the shared prompt and policy surface that future lanes must use to choose `remove fallback`, `expire fallback`, or `justify retaining fallback` for each touched fallback/seam.
- Default maximum lifetime: general repo fallback `60 days`; named high-churn control surfaces `30 days`; security/auth/PII/customer/financial/production-impact fallback `14 days`; external migration bridge `90 days` only with reviewer approval and a named owner issue.
