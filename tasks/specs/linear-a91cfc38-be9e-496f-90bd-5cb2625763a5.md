# TECH_SPEC - CO-555 exclude terminal retryable provider claims from active WIP

## Summary
- Objective: ensure terminal Linear issue truth excludes retryable/resumable provider-intake claims from active WIP, retry counts, selected active issues, and rehydration retry queues.
- Scope: provider-intake active/retry predicates, provider issue rehydration release logic, runtime/selected-run projections, and focused provider-intake/handoff tests.
- Constraints: no manual `provider-intake-state.json` edits, no CO-512 special-case branch, no deletion of retained audit history, and no weakening for non-terminal retry/resumable workers.

## Issue-Shaping Contract
- User-request translation carried forward: terminal issue metadata must beat `retry_queued`, `resumable`, and resume-eligible run status for active WIP.
- Protected terms / exact artifact and surface names: `provider-intake-state.json`, `provider_issue_rehydrated_resumable_run`, `retry_queued`, `resumable`, `issue_state=Done`, `issue_state_type=completed`, `active_issue_identifiers`, `isActiveProviderIntakeClaim`, terminal Linear truth, no manual state edits.
- Nearby wrong interpretations to reject: hand-editing intake state, relaunching CO-512, hiding the issue only in `co-status`, weakening non-terminal retry workers, or treating cached run status as stronger than terminal Linear truth.
- Explicit non-goals carried forward: no manual cleanup, no CO-512 relaunch, no retained-audit deletion, no broad control-host rewrite.

  - Preserve non-terminal retry/resumable workers as active and retry-visible.
- Non-functional requirements:
