# Technical Specification

## Context

`CO-306` is a focused follow-up on live provider-worker behavior after `CO-185`.

The live `CO-306` issue body says the exact helper-preflight failure modes `CO-185` claimed to contain are still present in Apr 22 active runs:

1. `CO-295` still records repeated `provider_worker_child_lane_parent_dirty` child-lane launch failures and invalidation churn during the same live attempt.
2. `CO-299` still records repeated same-attempt `linear_follow_up_parity_matrix_missing` failures and a parent-dirty child-lane launch failure in the same run.
3. `CO-302` still records repeated same-attempt `linear_follow_up_parity_matrix_missing` failures inside one live run.

This means the `CO-185` guarantees are either regressed or not actually being leveraged by the current provider-worker truth, prompting, or retry suppression path.

The issue is about restoring the live provider-worker helper-preflight contract. It is not permission to weaken parity-matrix enforcement, weaken clean-parent child-lane safety checks, broaden active issue scopes, or redesign scheduler/dispatch/concurrency policy beyond what is needed to restore the `CO-185` guarantees.

## Requirements

1. Reproduce and isolate the live Apr 22 failure shapes:
   - use `CO-295`, `CO-299`, and `CO-302` audit evidence from the current provider-worker path;
   - determine why the `CO-185` contract did not hold in live attempts.
2. Restore same-attempt suppression for deterministic parity-follow-up failures:
   - after `linear_follow_up_parity_matrix_missing`, unchanged same-attempt retries must not reissue the mutation;
   - allow retry only when inputs materially change, such as supplying the required parity matrix or changing the follow-up classification appropriately.
3. Restore deterministic parent-dirty sequencing/guidance:
   - preserve clean-parent checks;
   - prevent repeated `provider_worker_child_lane_parent_dirty` launch failures in the same attempt by surfacing actionable sequencing or recovery guidance.
4. Ensure the current live provider-worker path actually leverages the restored guarantee:
   - the first deterministic failure must materially change same-attempt worker guidance or retry behavior;
   - the fix must land in the seams used by live provider-worker attempts, not only in isolated helper code that current attempts do not consult.
5. Preserve guardrails while reducing retry churn:
   - do not weaken parity-matrix enforcement;
   - do not weaken clean-parent checks;
   - do not broaden scheduler, dispatch, or concurrency policy beyond this helper-preflight guarantee gap.
6. Add focused regression coverage:
   - prove same-attempt parity-follow-up suppression in the current provider-worker path;
   - prove parent-dirty child-lane situations converge through deterministic sequencing/guidance;
   - prove both guarantees hold without weakening the fail-closed contracts.
