# Technical Specification

## Context

`CO-67` replaces the narrower `CO-65` and `CO-66` split with one operator-facing fix for `CO STATUS`. The remaining paused defect is not the live alternate-screen design itself; it is the race where a live refresh already in flight can finish after pause and emit a second paused primary redraw. The remaining telemetry defect is not only formatting; it is a source-authority problem where compatibility runtime treats historical raw `in_progress` manifests as current work, inflates aggregate runtime, forces unavailable tokens to zero, and can ignore fresher live budget state when rendering rate limits.

This lane must keep the existing live `CO STATUS` monitor while making the paused handoff singular and making the top-level running/runtime/token/rate-limit summary authoritative.

## Requirements

1. Pausing from live monitoring must yield one deliberate paused `CO STATUS` snapshot in primary scrollback.
2. A live refresh that completes during the pause handoff must not append a second paused primary frame.
3. Current running rows must come from authoritative current activity rather than every historical manifest with raw status `in_progress`.
4. Aggregate runtime must exclude stale historical runs that are no longer authoritative.
5. Aggregate token totals must preserve unavailable values as unavailable rather than coercing them to zero.
6. Rate-limit summary must prefer authoritative live budget data when present and otherwise use the freshest authoritative worker proof.
7. Focused tests and screenshot proof must cover both the paused handoff truthfulness and authoritative telemetry corrections.

## Current Truth

- `startControlStatusDashboard(...)` already uses live alternate-screen rendering and a paused primary handoff, but `renderFrame(...useCachedFrame...)` can still emit a second paused `ANSI_CLEAR_HOME` frame after the initial exit-to-primary snapshot when pause races an in-flight live refresh.
- `discoverCompatibilityCollectionContexts(...)` scans historical `.runs/**/cli/*` manifests and currently pushes every discovered raw `in_progress` context into the compatibility `running` collection.
- `readCompatibilityRuntimeSnapshot()` then aggregates runtime and rate limits from those discovered contexts, even when the only authoritative current activity is the selected run plus active provider-intake claims.
- `buildCompatibilityTelemetrySnapshot(...)` zero-coerces missing token counts and only considers worker-proof rate limits.

## Proposed Design

### 1. Single paused primary handoff

- Preserve live alternate-screen rendering.
- Add a pause-handoff guard in `controlStatusDashboard.ts` so an already-started live refresh cannot append a second paused primary redraw immediately after the first handoff snapshot.
- Keep later explicit paused rerenders in-place on primary as they work today.

### 2. Authoritative current running sources

- Treat the selected run as authoritative current activity for its issue.
- Treat provider-intake claims as the authority for other currently active issues.
- Stop promoting arbitrary historical manifest-derived `in_progress` siblings into the current running collection unless they are the selected run or correspond to an active authoritative claim.
- Keep fallback manifest-only retry discovery when provider-intake state is absent.

### 3. Null-aware aggregate telemetry

- Aggregate runtime only across authoritative current running sources.
- Aggregate token totals with null-aware semantics:
  - sum only authoritative numeric values
  - keep each aggregate field null when no authoritative source reports that field
- Preserve row-level null token behavior already used in the running presenter.

### 4. Explicit rate-limit source preference

- Prefer live authoritative provider budget data from current host or polling state when available.
- Otherwise pick the freshest authoritative worker-proof rate limits across current running sources.
- Do not downgrade to `unavailable` when one of those authoritative sources exists.

### 5. Docs and proof

- Keep the docs-first packet aligned with the narrower operator-truthfulness scope.
- Capture screenshot proof directly in Linear covering:
  - live `CO STATUS` before pause
  - paused inspect state
  - scrollback after pause
  - corrected running/runtime/tokens/rate-limit summary
