## Technical Requirements
- Functional requirements:
  1. Add a read-only advisory to `doctor`, `co-status`, or a shared equivalent checkout-posture surface.
  2. Use local Git refs to compute ahead/behind counts against `origin/main` without network access.
  3. Classify at least:
     - stale: `behind > 0`
     - current: `ahead == 0` and `behind == 0`
     - ahead-only: `ahead > 0` and `behind == 0`
     - no-remote: `origin/main` or the configured main remote-tracking ref is unavailable
     - divergent: both `ahead > 0` and `behind > 0`, if encountered
  4. Emit dirty-worktree state separately from branch/worktree drift state.
  5. When the checkout is stale or divergent, include a warning that names the latest known posture commit/issue available from local current-main truth and tells the operator what current main believes about Codex CLI/model posture.
  6. Current packet drafting posture summary to preserve until parent implementation chooses a central metadata source:
     - local `origin/main` commit `632474813` / `CO-355 rebaseline marketplace smoke for Codex 0.125`
     - `CO-352` adopted CO-local ChatGPT-auth/appserver `gpt-5.5` / `xhigh` posture on Codex CLI `0.125.0` when live access smoke passes
     - release-facing package/downstream-smoke compatibility target is Codex CLI `0.125.0`
     - cloud execution candidate remains Codex CLI `0.124.0`
     - portable packaged/generated fallback defaults remain `gpt-5.4` / `xhigh`
  7. No-remote cases must fail soft with an explicit unknown comparison state and no destructive remediation.
- Non-functional requirements:
  - advisory is non-destructive
  - no network calls are required
  - output is concise enough for `doctor` / `co-status`
  - structured output, when available, remains machine-checkable
  - implementation is small and keeps posture summary centralized or tested
- Interfaces / contracts:
  - human-readable `doctor` / `co-status` output
  - JSON output for any affected `--format json` surface
  - local Git commands or library calls limited to read-only inspection
  - focused tests in the parent-owned checkout-posture test surface
