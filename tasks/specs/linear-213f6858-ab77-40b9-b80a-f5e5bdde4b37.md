## Summary
- Objective: Surface active Codex CLI and app-bundle CLI provenance in `codex-orchestrator doctor`.
- Scope: Doctor JSON/text output, Codex CLI provenance probing, focused tests, and task mirrors.
- Constraints: Read-only only; preserve `CODEX_CLI_BIN`, managed CLI, checkout-posture, and release-intake boundaries.

## Issue-Shaping Contract
- User-request translation carried forward: Doctor must make clear which active Codex executable was audited and whether the Codex.app bundled CLI differs.
- Protected terms / exact artifact and surface names: `command -v codex`, `/opt/homebrew/bin/codex`, `/Applications/Codex.app/Contents/Resources/codex`, `codex --version`, `CODEX_CLI_BIN`, `codex-orchestrator doctor`, active Codex binary, app bundle binary, binary provenance, version drift.

- Explicit non-goals carried forward: no binary replacement, no app/global CLI match requirement, no destructive cleanup, no version-promotion decision.

## Readiness Gate
- Not done if: Doctor reports only one implicit version; audited executable path is unclear; drift can be mistaken for checkout/docs posture; override or managed selection changes.
- Pre-implementation issue-quality review evidence: This spec preserves the Linear issue intent checksum, wrong interpretations, non-goals, and not-done-if clauses before source edits.
- Safeguard ownership split: CO-450 owns doctor binary provenance only; checkout-posture and release-intake remain linked lineage surfaces, not replacement scope.

## Technical Requirements
- Functional requirements:
  - Resolve and report the active Codex executable path for the selected command.
  - Execute `--version` on the active executable and report the normalized raw version string.
  - Detect `/Applications/Codex.app/Contents/Resources/codex` when present and report its `--version` separately.
  - Emit advisory drift when active and app-bundle versions differ.
  - Keep matching versions and absent app bundle non-noisy.
  - Preserve `CODEX_CLI_BIN` override precedence over managed selection.
- Non-functional requirements (performance, reliability, security):
  - Probes are read-only, bounded with short timeouts, and tolerate missing/unavailable binaries.
  - No filesystem mutation outside task docs and implementation files.
  - No sensitive environment values are emitted.
- Interfaces / contracts:
  - `DoctorResult.codex_cli` JSON extends without removing existing fields.
  - `formatDoctorSummary` text names active path/version, app bundle status/version, and version drift.
