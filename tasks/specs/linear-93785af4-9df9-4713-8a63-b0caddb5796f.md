# TECH_SPEC - CO-460 stale tracked.linear advisory fallback regression

last_review: 2026-05-03
status: active
owner: Codex
related:
- CO-223
- CO-255

## Contract
`co-status --format json`, `/api/v1/state`, and `/ui/data.json` must not project stale retained `linear-advisory-state.json` data as active top-level `tracked.linear`. When current authoritative tracked issue truth is unavailable, the shared projection must fail closed to `tracked.linear=null` with machine-readable stale/unavailable evidence such as stale-source metadata. A retained advisory issue can only become active tracked work when current source truth validates the same issue.

## Implementation Surface
- `orchestrator/src/cli/control/linearWebhookController.ts`: stale-source classification for retained advisory state when provider-intake/current control-host truth supersedes it.
- `orchestrator/src/cli/control/controlServerSeededRuntimeAssembly.ts`: restart/seed assembly path that marks stale advisory state before runtime snapshots.
- `orchestrator/src/cli/control/controlRuntime.ts`: top-level tracked projection must continue to suppress stale or conflicting advisory fallback.
- `orchestrator/src/cli/control/observabilitySurface.ts` and `orchestrator/src/cli/control/operatorDashboardPresenter.ts`: API/UI surfaces should remain aligned through the shared projection.
- Tests: `orchestrator/tests/ControlServerSeededRuntimeAssembly.test.ts`, `orchestrator/tests/ControlRuntime.test.ts`, and focused API/UI coverage where the projection shape matters.

## Required Behavior
- Fresh provider-intake/control-host state that no longer contains or validates the retained advisory issue marks the advisory state stale when it is newer than the advisory tracked reference.
- Existing matched-claim stale handling remains intact.
- Polling-only provider-intake heartbeats that do not advance issue truth must not falsely stale-mark an otherwise current advisory issue.
- `tracked.linear` must be null or current-authoritative on all three surfaces; stale `CO-1` must never reappear as active work.
- Stale-source metadata must identify this as provider-intake/current-truth supersession, separate from provider-intake summary drift.

## Regression Fixture
- Seed a retained `linear-advisory-state.json` tracked issue with `identifier=CO-1`, March 2026 `updated_at`, and accepted advisory result.
- Seed fresh provider-intake/control-host state with newer `updated_at` / `rehydrated_at` and no current validation for `CO-1`.
- Restart or assemble the control runtime from those seeds.
- Assert stale-source metadata exists, selected/top-level tracked projections are null, and API/UI datasets match.

## Not Done If
- Any status surface can display stale `CO-1` as top-level `tracked.linear`.
- The test only covers selected issue conflict suppression and not the retained-advisory restart case.
- Fresh unmatched provider-intake truth leaves old advisory state eligible as active tracked work.
- The change widens into provider-intake summary drift, binary/model provenance, or source-root freshness.
