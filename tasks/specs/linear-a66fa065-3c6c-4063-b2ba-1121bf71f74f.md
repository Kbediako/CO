## Summary
- Objective: make skipped-review summaries and feedback truthful when review is skipped by a failed prerequisite guard/stage.
- Scope: `TaskManager` skipped-review wording, `CommandBuilder` error artifact propagation, and focused tests.
- Constraints: preserve existing review result schema and existing true build/test failure wording.

## Parity / Alignment Matrix
- Current truth: failed prerequisite stages can arrive as failed build results and cause `Review skipped: build stage failed.`
- Reference truth: manifest `status_detail` and failed command records can identify the failed stage, such as `stage:delegation-guard:failed` plus a failed `delegation-guard` command.
- Target truth / intended delta: skipped-review summaries name the failed prerequisite stage when known, and include an error artifact path when the builder provides one.
- Explicitly out-of-scope differences: historical run-summary rewrites and unrelated provider closeout metadata.
