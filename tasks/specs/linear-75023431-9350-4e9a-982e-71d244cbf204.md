## Scope
- Make `linear child-lane --action accept` use and record the current issue snapshot at accept time.
- Require live current issue truth at accept time; unavailable or backwards `issue_updated_at` fails retryably instead of being marked stale.
- Preserve `provider_worker_child_lane_stale` for parent HEAD movement, live forward `issue_updated_at`, and semantic issue state/type drift.

- Keep base SHA stale checks and all patch/proof/scope/dirty checks unchanged.
- Add deterministic focused tests without live Linear calls.
