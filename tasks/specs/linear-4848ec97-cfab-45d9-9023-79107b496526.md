# Technical Specification

## Context

The autonomous provider-worker lane was recently aligned to current Symphony workflow states and workpad/PR behavior, but the current contract still stops short of the repo’s review discipline. Non-trivial implementation cycles in CO are supposed to run standalone-review and then elegance-review before handoff or merge, yet `providerLinearWorkerRunner.ts` only requires PR feedback sweep, validation, PR attachment, latest `origin/main`, and green PR checks before `Human Review` / `In Review`.

## Requirements

1. Add explicit standalone-review gating for non-trivial provider-worker diffs before PR create/update intended for review handoff.
2. Add explicit elegance/minimality gating after standalone-review findings are addressed and before review handoff for the same non-trivial diff.
3. Define the default non-trivial heuristic as about `2+ files` or about `40+ changed lines`, with explicit recorded justification required to skip.
4. Require the worker to record review goal, findings or fallback, and final clean or justified status in the single `## Codex Workpad` comment.
5. Define a manual fallback path when review tooling is unavailable: manual correctness/regressions/missing-tests review plus a manual elegance checklist, recorded in the workpad.
6. Cover the gate across both first-turn and continuation provider-worker prompt paths.
7. Keep the change bounded to the provider-worker prompt/docs/tests seam without altering unrelated provider features.
8. Keep the issue active until a PR is attached and the team review handoff is ready.

## Current Truth

- `orchestrator/src/cli/providerLinearWorkerRunner.ts` currently builds first-turn and continuation prompts that require state routing, workpad discipline, PR feedback sweep, PR attachment, latest `origin/main`, and green PR checks.
- The current prompt does not explicitly require standalone-review or elegance-review before review handoff.
- The current prompt does not define the non-trivial review-gate heuristic or a manual review/elegance fallback path.
- The current prompt does not require the workpad to record review goal, findings or fallback, and final clean or justified status before handoff.
- Current prompt assertions live in `orchestrator/tests/ProviderLinearWorkerRunner.test.ts`; workflow-helper behavior remains covered by `orchestrator/tests/ProviderLinearWorkflowFacade.test.ts`.
- Recent audit evidence is captured in `out/linear-4848ec97-cfab-45d9-9023-79107b496526/manual/20260325T065509Z-baseline-audit.md`.
- This worker run cannot use spawned subagents, so delegation guard must be satisfied with an explicit override rather than fabricated delegation evidence.
