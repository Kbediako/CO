- keep `AGENTS.md`, `.agent/AGENTS.md`, and `.agent/readme.md` aligned with that exact contract

## Current Repo Truth

- `package.json` maps `npm run test` to `test:core` with argument forwarding preserved.
- `test:core` runs `vitest run --config vitest.config.core.ts`.
- `test:all` runs `npm run test:core && npm run test:adapters`.
- `test:orchestrator` aliases `test:core`.
- `test:adapters` and `test:evaluation` use `vitest.config.ts`.
- `eval:test` aliases `npm run test:evaluation` with argument forwarding preserved.
- `.github/workflows/core-lane.yml` runs `npm run test:core` in its `Test (core matrix)` step.
- `AGENTS.md` documents `npm run test` in the required validation floor.
- `.agent/AGENTS.md` documents `npm run test:core`, the core/default `npm run test`, the explicit broader `npm run test:all`, and the opt-in `npm run eval:test`.
- `.agent/readme.md` lists `npm run test:core`, `npm run test`, and `npm run eval:test` in its build/test checklist.
