## Required Outcome

The repository must expose a truthful test matrix contract:

- Core Lane uses `npm run test:core`.
- `npm run test` is the default alias to `test:core`, not a repo-wide matrix.
- `test:all` is the explicit broader adapter-inclusive matrix.
- Adapter and evaluation commands use scoped Vitest configs so focused filters remain focused inside each lane.
- Evaluation remains opt-in through `test:evaluation` / `eval:test`.
- Docs and tests verify the contract rather than relying on implication.

## Acceptance Criteria

- [x] `package.json` includes explicit `test:core` and `test:all` commands.
- [x] Delegated aliases preserve npm-run argument forwarding.
- [x] `.github/workflows/core-lane.yml` invokes `npm run test:core`.
- [x] Agent docs distinguish `test:core`, `test`, `test:all`, and evaluation scope.
- [x] Regression coverage asserts the command/workflow mapping and focused adapter/evaluation filter behavior.
- [ ] Validation and review evidence are recorded before review handoff.

## Non-Goals

- No silent Core Lane expansion to `test:all`.
- No deletion or weakening of adapter/evaluation command surfaces.
- No generic CI cleanup or performance work.
