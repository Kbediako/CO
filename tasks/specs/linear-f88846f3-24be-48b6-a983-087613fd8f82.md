# TECH SPEC - CO-426 Provider Validation Audit Isolation
last_review: 2026-05-31
## Source / Intent
Linear `CO-426`; source issue `CO-405`; target surfaces include `orchestrator/tests/LinearCliShell.test.ts`, `orchestrator/src/cli/linearCliShell.ts`, and `co-status` audit projection. Prevent provider-worker validation from appending synthetic fixture rows such as `lin-issue-1` into a live `CODEX_PROVIDER_LINEAR_AUDIT_PATH` while preserving real provider helper audit records.
## Protected Terms
`CODEX_PROVIDER_LINEAR_AUDIT_PATH`, `provider-linear-worker-linear-audit.jsonl`, `lin-issue-1`, `LinearCliShell.test.ts`, `create-follow-up`, `linear_follow_up_parity_matrix_missing`, `co-status`, live provider-worker audit.
## Design / Regressions
Make mutating Linear CLI validation hermetic by clearing, overriding, or mocking the provider audit path in tests that use fixture issue ids. Add a regression that sets a live-looking `CODEX_PROVIDER_LINEAR_AUDIT_PATH`, runs the fixture path, and proves the live audit file is unchanged. Preserve normal audit appends for actual helper calls. If stale mismatched fixture rows can still be read from old artifacts, have `co-status` classify or ignore rows whose issue id cannot belong to the active provider issue instead of presenting them as latest live operations.
## Non-Goals
No weakening of parity-matrix guards, no removal of real provider helper audit logging, no CO-405 review-command-intent patch, and no generic hiding of provider audit failures.
## Not Done If
- `CODEX_PROVIDER_LINEAR_AUDIT_PATH` can still receive fixture rows from `LinearCliShell.test.ts`.
- `provider-linear-worker-linear-audit.jsonl` can still show synthetic `lin-issue-1` / `linear_follow_up_parity_matrix_missing` rows as live provider activity.
- The implementation weakens `create-follow-up` parity-matrix guards instead of making validation hermetic.
- Real provider helper audit logging for actual issue ids is removed or hidden.
- `co-status` still consumes mismatched fixture rows without classification or source filtering.
