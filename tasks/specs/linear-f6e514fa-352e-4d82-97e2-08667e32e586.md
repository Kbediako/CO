# Technical Specification

## Context

Current CO already has a durable Linear workpad contract from `CO-21`: one active `## Codex Workpad` comment, canonical section order, non-empty core sections, mirrored ticket validation requirements, and milestone-driven refresh guidance. The remaining gap is narrower and more operator-facing. Because section bodies are still free-form, `Acceptance Criteria` and `Validation` can be prose, plain bullets, or checkboxes, which weakens at-a-glance completion scanning during implementation and review handoff.

## Requirements

1. Enforce non-empty checkbox list items in the workpad `Acceptance Criteria` section.
2. Enforce non-empty checkbox list items in the workpad `Validation` section.
3. Keep `Environment / Workspace Stamp`, `Plan`, and `Notes` free-form as long as they remain non-empty and within the existing canonical section order.
4. Preserve the existing mirrored `Validation` / `Test Plan` / `Testing` requirement extraction so those mirrored requirements can still be satisfied inside checkbox-based acceptance or validation lists.
5. Update the provider-worker prompt so it describes the narrow checkbox-only contract instead of a broader checklist normalization.
6. Update the repo-local `skills/linear/SKILL.md` wording so workers and operators see the same narrow contract.
7. Add focused regressions covering:
   - acceptance of valid checkbox content
   - rejection of plain-bullet `Acceptance Criteria`
   - rejection of prose-only `Acceptance Criteria`
   - rejection of plain-bullet `Validation`
   - rejection of prose-only `Validation`
   - acceptance of free-form content in the other three sections
8. Keep the issue active until live Linear reads recover enough to refresh the workpad and state truthfully.
