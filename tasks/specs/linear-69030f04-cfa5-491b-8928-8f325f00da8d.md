# Technical Specification

## Context

`CO-55` made `CO STATUS` deliberately inspectable by adding pause/resume, compact inspect, snapshot export, and the dedicated `co-status` launch surface. The remaining problem in `CO-60` is narrower: the dashboard still renders every interactive frame into the primary terminal buffer. As a result, pausing after a few live refreshes still leaves scrollback full of historical `CO STATUS` frames. The current frame is truthful, but the paused inspection path is noisy because the operator is effectively reading redraw history instead of one deliberate snapshot.

This lane must keep `CO STATUS` as the same live monitor while changing the terminal-mode contract so normal live redraw does not pollute primary scrollback and pausing produces one clean inspect snapshot.

## Requirements

1. Interactive live `CO STATUS` redraw must stop stacking full historical frames in the primary terminal scrollback path.
2. Pausing must yield one clean static inspect frame in the primary terminal buffer.
3. While paused, explicit rerenders such as compact-mode toggles or snapshot-status updates must redraw in place instead of appending new full-frame history.
4. Resuming must return the operator to the same live dashboard surface without losing the existing pause/compact/snapshot controls.
5. JSON and non-interactive control-host output must remain unchanged.
6. Focused tests must fail closed if live redraw returns to primary-buffer frame stacking or if paused inspection falls back to stacked historical redraw dependence.
7. Docs and proof must explicitly describe and validate the chosen terminal semantics.

## Current Truth

- `startControlStatusDashboard(...)` writes every rendered frame as `${ANSI_CLEAR_HOME}${frame}`.
- Timer ticks call `requestRefresh()` and runtime subscriptions queue rerenders normally until paused.
- Pressing `p` today only toggles the `paused` state and rerenders the cached frame; it does not change the terminal buffer surface.
- Compact inspect and snapshot export already work on the dashboard state, so the missing seam is terminal-surface management rather than dashboard data or control discovery.

## Proposed Design

### 1. Live alternate-screen surface

When interactive TTY output is available, the dashboard should enter the terminal alternate screen before live rendering begins. Live redraws continue using the existing clear-and-home behavior, but they now occur inside the alternate screen instead of the primary scrollback path.

### 2. Paused primary-buffer snapshot handoff

When the operator presses `p` to pause:

- exit alternate screen
- render the current dashboard state once as a static snapshot into the primary terminal buffer
- leave that snapshot as the paused inspection surface the operator can scroll around

This is the one intentional paused snapshot that replaces the old redraw-history path.

### 3. Paused redraw-in-place behavior

Once paused on primary:

- timer and runtime-triggered updates remain suppressed, as `CO-55` already intended
- explicit inspect changes (`c`, snapshot-status updates) redraw the paused frame in place in primary rather than appending another full frame to scrollback
- pending-update state still accumulates until resume

### 4. Resume back to live monitor

When the operator presses `p` again:

- re-enter alternate screen
- trigger the existing resume logic, forcing a refresh if pending updates accumulated while paused
- continue normal live monitoring on the same `CO STATUS` surface

### 5. Docs and proof updates

Update README and the task packet so the operator-facing contract is explicit:

- live `CO STATUS` uses alternate-screen rendering
- paused inspection is a clean primary-buffer snapshot
- compact inspect and snapshot export remain available in that paused state
