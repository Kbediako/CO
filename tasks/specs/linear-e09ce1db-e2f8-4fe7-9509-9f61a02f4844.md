# Technical Specification

## Context
`CO-108` is not a general STATUS redesign. Shared Linear budget truth already exists in `linearBudgetState.ts`, the provider refresh lifecycle already records scheduling and active operation state in `providerPollingHealth.ts`, and `controlRuntime.ts` already persists and rehydrates the polling snapshot. The remaining bug is that operator-facing next-refresh truth still depends on raw `checking` and `next_poll_in_ms` fields, so the dashboard can contradict the active shared-budget cooldown window.

## Requirements
1. Under active shared-budget cooldown, `Next refresh` must match the authoritative cooldown or reset window rather than any stale scheduled countdown.
2. `checking now...` must appear only for a real in-flight poll attempt after cooldown suppression has cleared.
3. Ordinary scheduled polling must still render a countdown when there is no active cooldown and no real in-flight poll.
4. Requests-exhausted and complexity-exhausted paths must both preserve truthful next-refresh output.
5. Persisted polling snapshots must preserve truthful next-refresh output after restart or rehydrate.
6. Focused tests must cover cooldown countdown, cooldown-to-checking transition, and stale mismatch regression.

## Design
- Extend `ControlPollingHealthPayload` with projected next-refresh metadata derived from current raw polling fields plus shared-budget truth.
- Compute that projection in `providerPollingHealth.ts` with explicit precedence:
  1. active shared-budget cooldown
  2. real in-flight polling
  3. ordinary scheduled next poll
  4. unknown
- Preserve and backfill the new fields in `controlRuntime.ts` so persisted polling snapshots remain truthful even before the first live polling update.
- Update `compatibilityIssuePresenter.ts` and `controlStatusDashboard.ts` to consume projected next-refresh fields instead of inferring precedence locally from raw `checking` or `next_poll_in_ms`.
- Keep actual scheduling behavior in `linearBudgetState.ts` and `controlServerPublicLifecycle.ts` intact unless a narrowly scoped truth bug proves otherwise.
