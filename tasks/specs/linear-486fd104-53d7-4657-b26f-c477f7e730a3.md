# Technical Specification

## Requirements
1. Failed or degraded provider-worker lanes must read consistently across manifest, run-summary, out/state, and workpad closeout.
2. Closeout wording must preserve authoritative review outcomes, including `bounded-success`, `failed-boundary`, and `failed-other`.
3. Deterministic invalid provider mutations must not retry inside the same attempt.
4. Auxiliary proof harness runs must not appear as active provider work.
5. Focused regression coverage must reproduce the `CO-78` class.

## Design
- Shared truth helper: `orchestrator/src/cli/control/providerLinearWorkerTruth.ts`
- Closeout and telemetry freshness guard: `orchestrator/src/cli/services/commandRunner.ts`
- Selected-run/status projection override and proof-harness exclusion: `orchestrator/src/cli/control/selectedRunProjection.ts`
- Same-attempt mutation suppression guidance in worker prompts: `orchestrator/src/cli/providerLinearWorkerRunner.ts`
- Stale summary cleanup after later success: `orchestrator/src/cli/run/manifest.ts`, `orchestrator/src/cli/services/orchestratorLocalPipelineExecutor.ts`
