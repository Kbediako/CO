# Technical Specification

## Context

CO already matches Symphony on much of the provider-worker Linear lifecycle, but app-touching review proof is still missing. Current Symphony workflow requires runtime validation and media upload before review handoff; current CO workers have no comparable first-class command or documented handoff contract, and the permit template leaves video disabled by default. The missing seam is not broad "app automation"; it is a truthful, bounded runtime-proof capability surface that workers can discover, use, and audit before review handoff.

## Requirements

1. Add a worker-visible CO helper that evaluates runtime-proof policy for an app-touching lane and emits structured handoff content for Linear workpads and PR flow.
2. Make the helper fail closed when the requested proof mode is not allowed by permit policy or when no reviewer-usable handoff path is available.
3. Keep degraded capability explicit when screenshots or external proof links are allowed but video capture remains disabled.
4. Reuse or minimally extend the existing permit contract so screenshot, external-link, and video behavior are auditable from `compliance/permit.json`.
5. Update provider-worker prompt/help/docs so the new proof path is discoverable in the same workflow surface workers already use.
6. Add focused tests covering at least:
   - allowed screenshot or external-link handoff path
   - not-permitted path
   - video-disabled path
   - worker-visible prompt/help/documentation coverage
7. Keep the issue active until the docs-first packet, code, tests, and validation are complete and the PR/workpad handoff path is ready.
