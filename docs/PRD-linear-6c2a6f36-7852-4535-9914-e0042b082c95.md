## Summary
- Problem Statement: retained provider-intake claims can be locally `released` even while a same-issue provider worker is still live and discoverable from run artifacts, proof, or process evidence. Current rehydrate/refresh and CO STATUS projection can preserve or display stale released truth before evaluating the same issue's active run, leaving `provider-intake-state.json`, `counts.running`, and admission accounting behind live worker reality.
- Desired Outcome: when provider-intake reconciliation discovers same-issue live provider-worker evidence, that evidence takes precedence over stale retained `released` claim state. The claim becomes `running` with `reason: "provider_issue_rehydrated_active_run"`, active run identity, `run_manifest_path`, `worker_host`, and refreshed issue metadata when available. CO STATUS keeps the started worker visible while intake rehydrates. Released no-burn behavior from `CO-160` and `CO-162`, terminal pruning from `CO-182`, and historical non-provider pruning from `CO-73` remain intact when no same-issue live worker exists.
