## Scope
- Keep the lane bounded to provider-worker progress and stall telemetry, control-host claim/read-model projection, and authoritative per-issue debug snapshots.
- Reuse existing proof, claim, merge-closeout, and STATUS read surfaces instead of introducing a parallel artifact family.
- Preserve current behavior fixes from `CO-79`, `CO-80`, and `CO-81`; this lane only makes their live state visible and classifiable from one contract.

## Design
- Add a shared provider-observability helper that derives one structured progress and debug snapshot contract from provider-worker proof, intake claims, merge-closeout records, and live tracked-issue context.
- Extend `orchestrator/src/cli/providerLinearWorkerRunner.ts` so proof snapshots carry semantic progress metadata and emit mirrored semantic progress lines into the normal runner stream when that metadata changes.
- Extend selected-run, compatibility-issue, and operator-dashboard read models so the same authoritative debug snapshot is exposed through `co-status --format json`, `/ui/data.json`, `/api/v1/state`, and `/api/v1/<issue>`.
- Project merge-closeout and attached-PR state into the same snapshot instead of leaving that truth isolated inside `merge_closeout` claim records or worker-only PR monitors.
