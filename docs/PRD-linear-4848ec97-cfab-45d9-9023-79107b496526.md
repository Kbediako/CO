## Summary
- Problem Statement: CO repo policy already expects non-trivial implementation cycles to run a standalone review and then an elegance review before handoff or merge, but the autonomous provider-worker contract does not currently make either step an explicit pre-review-handoff gate. That leaves first-turn and continuation worker lanes free to open or update PRs and transition to `Human Review` or `In Review` without the repo’s intended review discipline.
- Desired Outcome: the provider-worker prompt/runtime contract explicitly requires a standalone review plus a paired elegance/minimality pass for non-trivial diffs before PR create/update intended for handoff and before transition to the live review state, while also documenting the manual fallback path and workpad recording expectations when tooling is unavailable.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): complete Linear issue `CO-11` in the current workspace by tightening the autonomous provider-worker prompt and supporting tests so the worker must run the repo’s standalone and elegance review cycle before review handoff, without widening into unrelated provider features.
- Success criteria / acceptance:
  - non-trivial provider-worker diffs require a standalone review before PR create/update intended for review handoff
  - after standalone findings are addressed, the worker runs an explicit elegance/minimality pass before review handoff
  - the heuristic default is roughly `2+ files` or about `40+ changed lines`, with an explicit recorded justification required to skip
  - the worker records review goal, findings or fallback, and final clean or justified status in the single `## Codex Workpad` comment
  - first-turn and continuation worker prompt/test coverage proves the gate is present across both paths
- Constraints / non-goals:
  - keep the change focused on autonomous pre-review-handoff review discipline
  - do not replace the existing PR feedback sweep, quiet-window handling, or merge-side gating
  - use the current Symphony baseline and current CO review policy as the authoritative contract
  - delegation must be recorded as an explicit override in this worker run because subagent spawning is unavailable in-session

## Goals
- Add explicit standalone-review gating to first-turn and continuation provider-worker instructions before review handoff.
- Add explicit elegance-review gating after standalone-review findings are addressed for the same non-trivial diff cycle.
- Require workpad recording of review goal, findings or fallback, and final review status before handoff.
- Make the fallback contract explicit when review tooling is unavailable so workers do manual correctness/regression/missing-tests review plus a manual elegance checklist instead of stalling.
- Cover the new review-gate contract in the provider-worker prompt/runtime tests.

## Non-Goals
- Changing merge shepherding behavior in `Merging`.
- Replacing the existing PR feedback sweep or PR-check green-bar requirements.
- Introducing new provider lifecycle states or broader worker capabilities beyond the review gate.
- Adding pre-implementation review approval requirements to the provider-worker prompt.
