# PRD - CO-489 External Agent Session Import Resume Continuity

## Summary
- Problem Statement: Codex added experimental external agent config and session import surfaces, including background imports and imported-session title handling. CO has provider-worker manifests, hook continuation, Linear workpads, and resume diagnostics, but it has not decided whether external agent session import belongs in that continuity model.
- Desired Outcome: CO-489 records the external-session import behavior from release/upstream evidence plus the current local app-server schema, classifies each CO continuity use case as adopt, defer, or reject, and keeps provider-worker manifests, provider-intake authority, hook continuation, and Linear workpads authoritative.

## Evaluation Result
- Decision: adopt external agent session import only as advisory diagnostics evidence, defer any CO automation until a fixture canary proves net-new operator value, and reject it for provider-worker resume authority, hook continuation authority, Linear state authority, and broad migration.
- Release evidence: OpenAI Codex `rust-v0.128.0` release notes published 2026-04-30 list external agent session import, background imports, and imported-session title handling.
- Current local evidence: `codex --version` reports `codex-cli 0.130.0`; `codex features list` reports `external_migration` as `experimental false`; `codex app-server generate-ts --enable external_migration` exposes `externalAgentConfig/detect`, `externalAgentConfig/import`, `externalAgentConfig/import/completed`, migration item type `SESSIONS`, and session fields `path`, `cwd`, and `title`.
- Upstream source/test evidence for `rust-v0.128.0`: session detection reads recent `.claude/projects/**.jsonl` records, requires an existing `cwd`, skips already-imported current source hashes through `external_agent_session_imports.json`, caps candidates at 50 sessions younger than 30 days, derives titles from custom title, ai title, or first user message, imports user/assistant messages into Codex rollout items, writes an `<EXTERNAL SESSION IMPORTED>` marker, and supports thread read/resume after import.
- Provenance conclusion: imported sessions preserve enough path/cwd/title/message/timestamp/import-marker evidence for diagnostics, but not enough CO lifecycle provenance to replace provider-worker manifests, run manifests, provider-intake records, hook continuation state, or Linear workpads.
