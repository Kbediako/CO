## Summary
- Problem Statement: `CO STATUS` still has two operator-facing truthfulness defects after the earlier inspectability and proof work. First, pausing during or around live refresh activity can still leave two visible paused `CO STATUS` frames in primary scrollback, so the paused handoff is not a true single-snapshot inspect state. Second, the top-level compatibility and dashboard telemetry remains non-authoritative because it can treat historical `.runs/**/cli/*` `in_progress` manifests as current activity, overstate runtime, coerce unavailable token telemetry to `0`, and surface `Rate Limits: unavailable` even when fresher budget or proof data exists.
- Desired Outcome: keep `CO STATUS` as the same live terminal monitor, but make the paused handoff a single deliberate primary snapshot and make the running/runtime/tokens/rate-limit summary derive only from authoritative current activity and authoritative live telemetry.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): fix `CO STATUS` as one coherent operator-facing lane rather than two narrow patches. Pausing must leave exactly one truthful paused snapshot in primary scrollback, and the dashboard summary must stop reporting historical or synthetic values as though they were current live truth.
- Success criteria / acceptance:
  - pausing from live monitoring yields one deliberate paused `CO STATUS` snapshot in primary scrollback
  - scrolling up after pause does not show a duplicate paused handoff copy of the same frame
  - running rows and aggregate runtime are derived from authoritative current activity rather than raw historical `in_progress` manifests
  - unavailable token telemetry renders as unavailable instead of `0`
  - rate-limit summary prefers authoritative live budget or proof data when available
  - final validation includes screenshot proof embedded directly in Linear covering live, paused, scrollback-after-pause, and corrected telemetry states
- Constraints / non-goals:
  - preserve live alternate-screen monitoring as the normal `CO STATUS` surface
  - do not reopen the broader `CO-44` parity or observability architecture
  - do not promise token telemetry where no authoritative source exists
  - do not satisfy screenshot validation with local file paths or proof indexes alone
