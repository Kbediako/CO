# PRD - CO: Embed screenshot proof directly in Linear comments and workpads

## Added by Bootstrap 2026-04-02

## Summary
- Problem Statement: `CO-55` required screenshot proof, but the resulting Linear-visible proof degraded into links or local artifact paths because the current provider-worker helper surface only supports workpad text plus PR or URL attachments. That is insufficient when a lane requires the actual screenshot to be visible inside Linear itself.
- Desired Outcome: give CO a direct server-side path that can take a local screenshot artifact, upload it into Linear storage, and embed the resulting image directly inside the active workpad comment so reviewers can see the proof without leaving the issue.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): finish `CO-61` by making screenshot proof mean an actual visible image in Linear, not a local file path, PR link, or external proof index. The solution should stay bounded to the minimum upload-and-embed path needed for local screenshot artifacts and should update the worker or workpad guidance so linked-only proof is no longer treated as sufficient when direct-in-Linear proof is required.
- Success criteria / acceptance:
  - a local screenshot artifact can be turned into a Linear-hosted asset URL through the worker-owned helper surface
  - the resulting proof is embedded directly inside a Linear issue context entry, with this lane using the single active workpad comment as the canonical proof surface
  - the audit trail records enough information to trace the upload and embed mutation, such as comment id, asset URL, or equivalent upload reference
  - guidance and templates explicitly distinguish embedded-in-Linear screenshot proof from external proof URLs
- Constraints / non-goals:
  - keep the change bounded to screenshot-proof upload and embed behavior
  - preserve existing PR attachment and external URL attachment flows
  - do not introduce a general binary asset platform or a broad new comment-management product surface
  - keep the proof path server-side and deterministic within the existing provider-worker helper architecture

## Goals
- Upload a local screenshot artifact into Linear-managed storage from the existing provider-worker helper surface.
- Embed the uploaded screenshot directly into the active workpad comment so the image is visible in Linear.
- Record audit and workflow evidence that identifies the resulting proof mutation.
- Update docs and worker guidance so `embedded directly in Linear, not only linked` is an explicit proof contract.

## Non-Goals
- Reworking unrelated PR attachment flows.
- Uploading every validation artifact to Linear.
- Building a general-purpose binary asset service or attachment browser.
- Treating wording-only guidance changes as sufficient without the upload-and-embed path.

## Metrics & Guardrails
- Primary Success Metrics:
  - at least one live validation example shows the actual screenshot embedded in the Linear workpad comment
  - the helper surface can convert a local screenshot file into a Linear-hosted asset reference without external proof hosting
  - workpad or workflow wording explicitly requires embedded-in-Linear proof when the lane calls for it
- Guardrails / Error Budgets:

  - maintainer audits the resulting mutation trail and can trace the proof to the updated workpad comment plus the uploaded asset reference

## Technical Considerations
- Architectural Notes:
  - the existing worker-owned Linear mutation substrate already owns workpad upserts, issue transitions, PR attachment, and runtime-proof policy
  - the narrowest truthful extension is:
    - request a Linear `fileUpload` URL from the server-side workflow facade
    - `PUT` the local screenshot bytes to that signed URL
    - rewrite the workpad markdown to reference the returned Linear `assetUrl`
    - preserve the existing single-workpad contract and audit trail
  - external proof URL guidance remains relevant for app-touching runtime-proof lanes, but this issue needs an additional path for local screenshot artifacts when proof must be visible directly in Linear
