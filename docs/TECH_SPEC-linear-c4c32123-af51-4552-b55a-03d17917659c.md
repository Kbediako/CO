## Summary
- Objective: add a simple read-only attach viewer for an already-running local JSON `control-host`, with live terminal scrollback available while the viewer stays live.
- Scope:
  - docs-first registration for `CO-74`
  - CLI/help updates for `co-status attach`
  - attach-side host resolution from persisted control-host endpoint/auth artifacts
  - read-only dataset fetch plus live primary-scrollback rendering for the attach viewer
  - focused regressions and operator docs/proof
- Constraints:
  - preserve the existing launched `co-status` host flow
  - do not start another provider polling loop in the attach viewer
  - keep the transport bounded to the local authenticated dashboard-read surface

## Technical Requirements
- Functional requirements:
  - `co-status attach` must exist as a discoverable CLI surface
  - attach must resolve the local host from persisted run-dir endpoint/auth artifacts and documented explicit task/run/run-dir flags
  - attach must fetch only the read-only dashboard dataset surface from the host
  - attach must not launch a new `control-host` or start another Linear/Telegram polling loop
  - live attach rendering must keep scrollback available without pause-first behavior
  - local inspect affordances (`p`, `c`, `s`) should remain available where they stay truthful and read-only
  - help/docs must explain launch vs attach and the artifact-resolution contract
- Non-functional requirements (performance, reliability, security):
  - artifact resolution must fail closed when the endpoint/auth files are missing or invalid
  - attach should stay loopback/local-host bounded by the same control endpoint validation posture
  - read-only attach should tolerate periodic polling/fetch failure with explicit viewer error framing rather than crashing silently
- Interfaces / contracts:
  - CLI entrypoint: `bin/codex-orchestrator.ts`
  - attach shell / endpoint loader: new or extracted CLI helper under `orchestrator/src/cli/**`
  - dashboard renderer: `orchestrator/src/cli/control/controlStatusDashboard.ts`
  - dashboard read route: `orchestrator/src/cli/control/uiDataController.ts`

## Architecture & Data
- Architecture / design adjustments:
  - add a dedicated `co-status attach` subcommand while preserving `co-status` default launch behavior
  - resolve the host run dir from `task` + `run` defaults or explicit `run-dir`, then load the persisted control endpoint/auth metadata
  - fetch the authenticated `/ui/data.json` payload and render it through the existing `CO STATUS` renderer or a narrow shared viewer abstraction
  - keep launched mode on alternate-screen live semantics, but keep attached live mode on primary scrollback semantics
  - keep attach input handling local-only so pause/compact/snapshot affect only the viewer process
- Data model changes / migrations:
  - no host-side data model changes are expected
  - if a dashboard-local surface flag is needed to distinguish alternate-screen launch mode from primary-scrollback attach mode, keep it renderer-local
- External dependencies / integrations:
  - persisted `manifest.json`, `control_endpoint.json`, and `control_auth.json` in the control-host run dir
  - authenticated `GET /ui/data.json` on the local host
