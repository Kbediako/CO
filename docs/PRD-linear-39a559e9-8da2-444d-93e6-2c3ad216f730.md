## Summary
- Problem Statement: Direct `node dist/bin/codex-orchestrator.js co-status --format json` can fail on a healthy control-host when the current resolved `/ui/data.json` endpoint hangs beyond the 15s attach timeout budget. The current direct JSON recovery path only retries when endpoint/auth artifacts change; a same-endpoint hang currently surfaces as a raw timeout even when local intake truth is healthy.
- Desired Outcome: the direct JSON path gains bounded same-endpoint timeout recovery or truthful classification, preserves the existing stale/dead-endpoint recovery semantics from `CO-246`, and keeps current-endpoint hangs distinct from attach restart/rotation failures and refresh-stuck churn.

## Technical Considerations
- Architectural Notes:
  - `coStatusCliShell.ts` owns the direct `co-status --format json` flow.
  - `coStatusAttachCliShell.ts` owns fetch/recovery helpers including `fetchUiDataset` and `readUiDatasetWithEndpointRecovery`.
  - The current recovery contract re-resolves endpoint/auth artifacts and only retries after endpoint/token change, which is insufficient for a current endpoint whose `/ui/data.json` request itself times out.
  - The likely minimal repair is to add one bounded same-endpoint timeout retry/classification branch to the shared direct JSON recovery helper, preserving retry limits and existing stale endpoint semantics.
- Dependencies / Integrations:
