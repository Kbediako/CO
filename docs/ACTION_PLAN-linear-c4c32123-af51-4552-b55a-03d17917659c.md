## Summary
- Goal: land the smallest truthful attach flow that lets operators read live `CO STATUS` from an already-running local JSON host without launching another host and without losing live scrollback.
- Scope: docs-first packet, audited docs-review child stream, `co-status attach` CLI/help, artifact-backed host resolution, read-only attach viewer rendering, focused tests/docs, real local proof, and the normal validation/review gates.
- Assumptions:
  - the running host's authenticated `/ui/data.json` surface is sufficient for the attach viewer dataset
  - the persisted `control_endpoint.json` + `control_auth.json` artifacts are the right local host-resolution seam
  - a narrow renderer/viewer generalization is acceptable if it stays local to the dashboard surface

## Risks & Mitigations
- Risk: the attach viewer accidentally triggers host refresh/mutating behavior.
  - Mitigation: limit attach to authenticated read-only dataset fetches and keep input handling local-only.
- Risk: a renderer tweak for attach regresses the launched `co-status` path.
  - Mitigation: keep launch-vs-attach surface mode explicit and add focused regressions for both semantics.
- Risk: artifact resolution becomes too permissive or guesses routes.
  - Mitigation: validate the persisted endpoint/auth contract and fail closed with explicit help.
- Risk: live primary-scrollback rendering leaves the shell prompt glued to the last frame on exit.
  - Mitigation: add focused primary-surface shutdown coverage in the dashboard tests if the implementation needs renderer-local newline handling.
