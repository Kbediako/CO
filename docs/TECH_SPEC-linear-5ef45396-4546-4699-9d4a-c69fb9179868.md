## Fallback Expiry Decisions

| Fallback / seam | Decision | Parent implementation guidance |
| --- | --- | --- |
| Appserver preflight failure auto reroutes to CLI | expire fallback | Retain only with CO-396 owner metadata, review date `2026-05-10`, maximum lifetime `2026-05-26`, removal condition, and focused activation/non-activation tests. |
| Explicit or strict cloud appserver runtime block | justify retaining fallback | Retain as durable fail-fast blocked metadata; do not allow explicit `executionMode=cloud` plus `runtimeMode=appserver` to launch a fallback target. |
| Implicit default appserver under cloud auto coerces to CLI | expire fallback | Retain only as a temporary bridge; later implementation should either select `cli` explicitly before runtime selection or fail fast with equivalent metadata. |
| `CODEX_ORCHESTRATOR_RUNTIME_FALLBACK` and `allowFallback` aliases | justify retaining fallback | Retain as durable compatibility only because all aliases normalize to explicit `auto` or `strict` with `policy_source`. |
| `RuntimeFallbackMetadata` for no-fallback, fallback, and blocked decisions | justify retaining fallback | Retain as durable audit schema for runtime decision traceability. |
| Runtime-mode canary fallback scenarios | justify retaining fallback | Retain as durable validation contract even if production fallback behavior narrows. |
