# PRD - CO Add Bounded Pre-Review Automated-Feedback Drain Before In Review

## Summary
- Problem Statement: the current CO provider-worker contract requires a PR feedback sweep before handoff, but it still allows the worker to move an issue into `In Review` without an explicit bounded automated-feedback drain after the latest PR update. That creates avoidable `In Review -> Rework` churn when CodeRabbit, the Codex connector, or required check feedback arrives shortly after the handoff.
- Desired Outcome: add a bounded, machine-checkable pre-review drain that keeps the issue in an active coding state until automated review/check feedback is either quiet, handled, explicitly pushed back, or timed out truthfully.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): complete `CO-10` in this workspace by codifying the emerging worker behavior from the live CO-6 lane into a shipped CO contract, while staying truthful to the current Symphony baseline and CO's current worker lifecycle.
- Success criteria / acceptance:
  - after a PR is opened or updated, the worker waits a bounded window for automated reviewer/check feedback before transitioning to `In Review`
  - actionable automated feedback on the current PR head is either addressed, explicitly pushed back with evidence, or treated as a blocker that keeps the issue out of `In Review`
  - a clean no-feedback timeout path exists when required checks are green and no actionable machine feedback appears during the window
  - late feedback after `In Review` still routes through the existing `Rework` contract
  - tests cover the clean no-feedback path, actionable-feedback-handled path, and bounded-timeout path
- Constraints / non-goals:
  - keep the wait bounded and auditable; do not add indefinite polling before review handoff
  - preserve the current stop-coding semantics once the issue reaches `In Review`
  - preserve the current `Rework` and `Merging` lifecycle boundaries from the existing Symphony parity work
  - audit the local Symphony checkout first and record the exact current baseline before implementation
  - delegation must be explicitly overridden in this worker run because spawned subagents are unavailable in-session

## Goals
- Add a shipped pre-review PR monitor mode that waits for green required checks plus a bounded quiet window before human-review handoff.
- Keep automated-review blockers explicit before `In Review`, including unresolved review threads, actionable bot inline comments, and equivalent machine-review findings on the current head.
- Update the provider-worker prompt and repo-local Linear skill so workers use the bounded drain before review handoff instead of relying on informal practice.
- Cover the new behavior with focused tests on the PR monitor, CLI surface, and provider-worker prompt contract.

## Non-Goals
- Reworking the post-human-review merge shepherding flow.
- Adding autonomous continuation beyond the current `In Review` stop boundary.
- Introducing a new workflow engine separate from the existing PR monitor and provider-worker prompt surfaces.

## User Experience

  - worker sees no actionable automated feedback before the bound expires, records the clean wait in the workpad, and transitions normally to `In Review`
