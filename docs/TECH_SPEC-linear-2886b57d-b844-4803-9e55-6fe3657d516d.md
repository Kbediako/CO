## Required Behavior
- `initialize` remains authority-neutral and continues to advertise only the existing tools capability.
- `tools/list` returns the same tool names per `full`, `question_only`, and `status_only` modes.
- Tool descriptors add only supported `title` and MCP `annotations` fields.
- Existing `inputSchema` objects remain unchanged.
- Read-only tools are annotated with `readOnlyHint: true`.
- Mutating tools are annotated with `readOnlyHint: false`.
- Destructive or confirmation-required tools are annotated with `destructiveHint: true`.
- Additive mutating tools are annotated with `destructiveHint: false`.
- `delegate.question.poll` remains non-read-only because expired polls can apply the configured fallback action to the child run.
- No per-tool parallel-call field is added; Codex 0.121 uses server-level `mcp_servers.<id>.supports_parallel_tool_calls`.

## Non-Goals
- No new tools or GitHub operations.
- No change to delegation mode boundaries.
- No change to dynamic bridge attestation policy.
- No change to confirm-nonce injection or model-secret rejection.
- No provider-worker runtime migration.
- No Apps widget `_meta` or marketplace distribution metadata.
