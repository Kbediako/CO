## Summary
- Problem Statement: `.runs/local-mcp/cli/control-host/provider-intake-state.json` can retain multiple concurrent running claims, while top-level `provider_intake` is still reduced to a single claim through `buildProviderIntakeSummary()` and `selectProviderIntakeClaim()` using `state-rank plus updated_at sort`. That singular summary can diverge from concurrent running claims visible in `CO STATUS` / `co-status --format json` and can hide relevant `retry/error history` when the non-selected claim is the one that still carries it.
- Desired Outcome: parent implementation establishes one truthful contract between top-level `provider_intake`, concurrent running claims, `CO STATUS`, `co-status --format json`, and retained `retry/error history` so operators can see concurrent work without a single summary claim implying a contradictory current truth.
