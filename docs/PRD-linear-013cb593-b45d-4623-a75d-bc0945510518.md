## Summary
- Problem Statement: CO operators can work from a branch or worktree whose local docs and posture guidance have drifted from the local `origin/main` reference. Existing dirty-worktree signals do not clearly tell the operator whether they have uncommitted local work, stale branch/worktree posture drift against main, or both. That ambiguity is dangerous for Codex CLI/model posture because current main may already encode newer posture decisions than the local checkout.
- Desired Outcome: parent implementation adds a read-only `doctor`, `co-status`, or equivalent advisory that reports ahead/behind counts against `origin/main`, warns when the checkout is behind current main posture truth, names the latest known posture commit/issue, states what current main believes about Codex CLI/model posture, and distinguishes dirty local work from stale branch/worktree drift without fetching, rebasing, checking out, or discarding anything.
