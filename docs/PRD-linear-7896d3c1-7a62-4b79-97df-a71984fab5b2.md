## Summary
- Problem Statement: runtime currently synthesizes `Guardrails: spec-guard command not found.` when no `spec-guard` stage exists. That false guardrail absence can bleed into manifest summaries, metrics closeout, provider issue observability, and retry truth, making operators or automation treat a non-existent guardrail failure as current run truth.
- Desired Outcome: the missing-guardrail summary appears only when a `spec-guard` stage was actually part of the run contract and is missing or failed. Runs with no `spec-guard` stage keep truthful manifest, metrics, observability, and retry evidence without synthesized guardrail-failure noise.
