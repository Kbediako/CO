## Risks & Mitigations
- Risk: active-run precedence revives a genuinely released historical run.
  - Mitigation: only prefer same-issue `status: "in_progress"` runs that are attachable under current provider pipeline/run discovery rules.
- Risk: released no-burn paths start doing direct issue-by-id reads again.
  - Mitigation: keep the new path limited to discovered live worker evidence and rerun prior no-burn regressions.
- Risk: live worker cleanup/cancel logic treats the active worker as a released cleanup target.
  - Mitigation: order active-run rehydrate before release cancel/cleanup and assert duplicate launch/cancel is not used as recovery.
- Risk: issue metadata remains stale after rehydrate.
  - Mitigation: reuse the existing rehydrate metadata-refresh path when `service.rehydrate()` is called.
