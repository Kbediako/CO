## Summary
- Objective: evaluate whether Codex external agent session import should become part of CO resume continuity, diagnostics, or migration workflows.
- Scope:
  - six CO-489 packet/mirror files

- Constraints:

## Evaluation Decision
- Adopt: diagnostics-only use when an operator needs advisory context from an external-agent-originated session. The imported session can contribute path/cwd/title/message/timestamp/import-marker evidence to a manifest or Linear workpad note.
- Defer: any CO automation that imports fixture sessions into task artifacts. That needs a later temp-`CODEX_HOME` fixture canary proving net-new value over existing provider-worker manifests.
- Reject: provider-worker resume authority, provider-intake authority, hook continuation authority, Linear transition authority, and broad historical migration.
- Follow-up decision: no implementation follow-up created from this lane because the evidence proves diagnostic usefulness but not a concrete canary-backed CO automation gap.

## Evidence Summary
- Release evidence: OpenAI Codex `rust-v0.128.0` release notes published 2026-04-30 list external agent session import, background imports, and imported-session title handling.
- Current local evidence: `codex --version` reports `codex-cli 0.130.0`; `codex features list` reports `external_migration` as `experimental false`; `codex app-server generate-ts --enable external_migration` exposes `ExternalAgentConfigDetectParams`, `ExternalAgentConfigImportParams`, `ExternalAgentConfigImportResponse`, `ExternalAgentConfigImportCompletedNotification`, and `SessionMigration`.
- Protocol shape: `externalAgentConfig/detect` accepts `includeHome` and `cwds`; `externalAgentConfig/import` accepts `migrationItems`; import completion is reported as `externalAgentConfig/import/completed`; session migration details carry `path`, `cwd`, and nullable `title`.
- Upstream `rust-v0.128.0` source/test behavior: session detection scans recent `.claude/projects/**.jsonl`, filters to existing cwd roots, skips already-imported current source hashes, caps candidates at 50 sessions younger than 30 days, prefers custom title over ai title over first user message, converts user/assistant messages into Codex rollout items, appends `<EXTERNAL SESSION IMPORTED>`, records imported source hashes in `external_agent_session_imports.json`, and supports thread read/resume after import.
- Provenance boundary: imported sessions preserve enough evidence for advisory diagnostics, but not CO run manifest lineage, provider-intake admission truth, child-lane lineage, PR feedback state, hook continuation state, or Linear workpad authority.
