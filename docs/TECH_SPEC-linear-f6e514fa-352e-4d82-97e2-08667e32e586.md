  - docs-first registration for `CO-30`

## Technical Requirements
- Functional requirements:
  - `upsert-workpad` must reject workpad bodies when `Acceptance Criteria` contains no non-empty checkbox list items
  - `upsert-workpad` must reject workpad bodies when `Validation` contains no non-empty checkbox list items
  - workpad bodies must still accept free-form non-empty content in `Environment / Workspace Stamp`, `Plan`, and `Notes`
  - prompt text in `providerLinearWorkerRunner.ts` and repo-local wording in `skills/linear/SKILL.md` must explicitly describe the narrow checkbox requirement instead of implying broader checklist normalization
  - ticket-provided `Validation`, `Test Plan`, or `Testing` requirements must still be mirrorable inside the workpad `Acceptance Criteria` or `Validation` sections
- Non-functional requirements (performance, reliability, security):
  - keep validation deterministic and local to the existing helper boundary
