## Summary
- Problem Statement: CO-21 tightened the Linear workpad contract around one persistent `## Codex Workpad`, canonical section order, non-empty sections, and ticket-validation mirroring, but it intentionally left section body shape flexible. In practice that means the operator-facing `Acceptance Criteria` and `Validation` sections still drift between plain bullets, prose, and checklist items, which makes completion state harder to scan during implementation and review handoff.
- Desired Outcome: CO should require non-empty checkbox list items only where the operator benefit is clear: `Acceptance Criteria` and `Validation`. The other canonical workpad sections should remain free-form so we do not overfit CO to Symphony's full nested checklist style.
