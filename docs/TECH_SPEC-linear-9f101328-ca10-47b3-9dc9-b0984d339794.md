## Summary
- Objective: make `CO STATUS` and `/ui/data.json` ignore accepted pending-revalidation rows as current running activity when their only local provider proof is a fresh dead PID on a stale `in_progress` run.
- Scope:
  - active-running classification in the compatibility/runtime projection
  - `running/max_allowed` output and `/ui/data.json` `counts.running`
  - stale PID/event suppression on active running surfaces
  - focused regressions that separate this accepted dead-proof shape from `CO-125`, `CO-162`, `CO-192`, and `CO-194`
- Constraints:
  - keep `max_concurrent_agents=3` unchanged
  - preserve genuinely live accepted pending-revalidation behavior
  - preserve released-row stale handling already covered by adjacent issues
