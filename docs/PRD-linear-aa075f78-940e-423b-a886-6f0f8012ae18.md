## Problem
After PR #838 merged for CO-558, Linear briefly made the docs freshness owner terminal. The operator restored CO-558 to a non-terminal owner posture and then moved it to `Backlog` so the retained `docs:freshness:maintain` cohort keeps a live same-project owner without consuming an active worker slot.

Provider-intake correctly released the old worker as `provider_issue_released:not_active` with `issue_state=Backlog`, `issue_state_type=backlog`, and `retry_queued=null`, but `co-status --format json` can still surface the stale failed run summary as current failed work. That creates false queue pressure and confuses passive owner state with active retry/recovery state.

## Goal
Make status projection distinguish passive released owner claims from active retry/failed worker work while preserving old audit artifacts and real active failures.

## Acceptance Criteria
- [ ] `co-status --format json` distinguishes passive released owner claims from active retry/failed work.
- [ ] Released non-terminal `Backlog` owner issues with `retry_queued=null` do not consume WIP and do not display as current failed work solely due to historical run summaries.
- [ ] Existing terminal retry/resumable exclusion behavior remains covered.
- [ ] Regression fixtures cover the CO-558 shape: merged PR, owner issue restored to Backlog, provider-intake released/not_active, stale failed run summary present.
- [ ] Operator guidance records when a merged docs-freshness owner should remain Backlog until the cohort is refreshed, archived, reclassified, or re-homed.

## Intent Checksum
Protected terms: `provider-intake-state.json`, `co-status --format json`, `provider_issue_released:not_active`, `Backlog`, `docs:freshness:maintain`, `passive owner issue`, `retrying`, `failed run summary`, `run_summary`, `retry_queued=null`.

Reject fixes that delete audit history, weaken retry semantics for genuinely active non-terminal workers, hide real failed active runs, or require manual JSON edits / control-host restarts as the accepted path.

## Non-Goals
- Do not weaken active retry/resumable worker recovery.
- Do not make terminal issues appear live just to satisfy docs freshness.
- Do not delete provider-intake audit rows or old run manifests.
- Do not change the docs freshness owner verification contract except where needed for status projection clarity.
- Do not broaden into provider admission, Linear lifecycle automation, or manual control-host recovery.

## Not Done If
- A released `Backlog` owner issue with `retry_queued=null` still appears as active, retrying, or failed in CO status because of an old run summary.
- Real failed active runs are hidden.
- The solution requires operator restarts or manual `provider-intake-state.json` edits.
- `docs:freshness:maintain` owner issues become terminal as part of closeout.
