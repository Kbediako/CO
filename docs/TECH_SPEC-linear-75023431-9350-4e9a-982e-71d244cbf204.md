## Summary
- Objective: make `linear child-lane --action accept` use and record current issue truth so backwards timestamp inversions fail retryably instead of as `provider_worker_child_lane_stale`.
- Scope and protected boundaries are canonical in the PRD; implementation stays in the child-lane accept helper, current snapshot evidence, and focused regressions.
- Not done if backwards current issue inversion is stale, current snapshot evidence is absent, real issue drift is accepted, or parent HEAD/patch/proof/scope/dirty guards weaken.

## Technical Requirements
- Functional requirements:
  1. Add focused regressions for the CO-486 and CO-493 backwards timestamp examples; expected result is a retryable current-issue diagnostic, not `provider_worker_child_lane_stale`.
  2. Add a preservation regression where current issue `issue_state` or `issue_state_type` differs from launch snapshot; expected result remains `provider_worker_child_lane_stale`.
  3. Add a preservation regression where live current issue `updated_at` moves forward; expected result remains `provider_worker_child_lane_stale`.
  4. Adjust stale-decision logic so parent run-manifest fallback is never treated as current issue truth and backwards timestamps are not stale.
  5. Persist or expose accept-time current issue snapshot evidence in the child-lane decision/proof/ledger path with enough fields to audit source, `issue_updated_at`, `issue_state`, `issue_state_type`, capture time, and unavailable reason.
  6. Keep `provider_worker_child_lane_stale` messages explicit for base SHA movement, forward live timestamp movement, and issue state drift.
  7. Keep patch artifact, proof bundle, scope contract, parent dirty overlap, missing artifact, and apply failure behavior unchanged.
- Non-functional requirements: smallest local change in the child-lane accept helper, deterministic tests without live Linear calls, no new lifecycle mode unless the schema cannot carry accept snapshot evidence, and human-readable/machine-readable stale reasons.
- Interfaces / contracts: `provider-linear-child-lane.patch` and `provider-linear-child-lane-proof.json` remain required; `parent_snapshot` remains launch provenance; accept-time current issue snapshot evidence is additive.
