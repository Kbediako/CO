## Parity / Alignment Matrix
- Current truth:
  - `getProviderLinearIssueContext(...)` reads Linear issue context, then calls `hydrateIssuePullRequestAttachments(...)`.
  - `hydrateIssuePullRequestAttachments(...)` dedupes GitHub PR URLs, then awaits each `resolvePullRequestSnapshot(...)` call serially.
  - `fetchIssueContextPullRequestSnapshot(...)` shells out to `gh pr view --json state,mergedAt,updatedAt`.
  - `classifyIssuePullRequestAttachments(...)` owns classification semantics after snapshots are resolved.
- Reference truth:
  - each unique PR snapshot read is independent after attachment URL dedupe.
  - concurrency can improve latency only if it remains bounded and preserves resolver-result ordering semantics where downstream classification depends on the resolved candidate set.
  - per-candidate failures belong in `unknown`, not as full issue-context failures.
- Target truth / intended delta:
  - the resolver calls run through a small bounded scheduler.
  - output classification is unchanged for the same resolved/unknown candidate set.
  - tests prove both the cap and the semantic parity.
- Explicitly out-of-scope differences:
  - `CO-220` correctness-scope changes
  - merge-closeout/review-promotion selection changes
  - unbounded fanout
  - GitHub API redesign
  - Linear attachment cleanup

## Technical Requirements
1. Keep URL parsing and duplicate GitHub PR URL suppression before scheduling snapshot work.
2. Resolve unique PR snapshots through an explicit bounded-concurrency helper or equivalent local scheduler.
3. Keep the default cap small and deterministic. A cap of `4` is acceptable unless parent evidence supports a different conservative value.
4. Preserve the injected resolver contract so tests can provide `resolvePullRequestSnapshot(...)` without shelling out to `gh`.
5. Preserve `fetchIssueContextPullRequestSnapshot(...)` output mapping from `state`, `mergedAt`, and `updatedAt`.
6. Preserve per-candidate `unknown` fallback when the resolver throws or returns an unmappable snapshot.
7. Preserve `classifyIssuePullRequestAttachments(...)` semantics and returned `pull_request_attachments` shape.
8. Keep the cached `provider-linear-issue-context-cache.json` shape compatible with current readers.
