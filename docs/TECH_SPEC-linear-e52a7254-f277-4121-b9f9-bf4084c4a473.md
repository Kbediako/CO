## Summary
- Objective: replace the current selected-run-oriented `/ui` dataset with a richer operator dashboard surface aligned with Symphony's terminal/dashboard information density while staying read-only and presenter-driven.
- Scope:
  - docs-first registration for the current Linear worker issue
  - a shared operator dashboard presenter over the existing compatibility projection and issue payloads
  - additive refresh/poll health surfacing from the control-host public lifecycle
  - `/ui` dataset and frontend updates to render the richer operator surface
  - focused presenter/controller/UI regression coverage plus required repo validation
- Constraints:
  - keep UI/API layers thin over shared presenter/read-model seams
  - preserve the existing read-only compatibility API and refresh semantics
  - do not reopen runtime ownership, retry scheduling, or tracker mutation logic
  - record the delegation override explicitly because this worker run cannot spawn subagents

## Technical Requirements
- Functional requirements:
  - `/ui` must show running sessions, retry queue, aggregate token/runtime totals, latest rate limits, and refresh/poll health from authoritative runtime-adjacent state
  - `/ui` must provide per-issue drilldown with identifier, lifecycle state, session/thread id when present, turn count, recent agent activity, workspace path/host, retry status, and last error when present
  - issue details must reuse the compatibility issue payload builder instead of creating a second issue-detail derivation path in the frontend
  - a shared operator-dashboard presenter must sit between UI/controller code and the compatibility projection
  - if refresh is exposed in the dashboard, it must delegate to the existing operational refresh path and remain non-correctness-bearing
  - additive poll-health data should be exposed to the shared presenter using the current control-host public lifecycle rather than a new orchestrator subsystem
- Non-functional requirements (performance, reliability, security):
  - preserve the current authenticated/public route split and session-token flow for `/ui`
  - keep the implementation read-only and avoid widening the control action surface
  - keep static asset serving and operator rendering robust when data is missing or partially unavailable
  - keep the change reviewable by reusing existing payloads and helper seams where possible
- Interfaces / contracts:
  - shared operator presenter contract: consumes `ControlCompatibilityProjectionSnapshot` plus optional polling-health state and returns one dashboard payload for `/ui/data.json`
  - issue-detail contract: reuses `CompatibilityProjectionIssueRecord.payload` / `ControlIssuePayload`
  - poll-health contract: exposes at least `checking`, `queued`, `interval_ms`, and `next_refresh_in_ms` when the provider handoff lifecycle is available
  - API contract: existing `/api/v1/state` and `/api/v1/:issue_identifier` remain read-only and may be extended additively for poll-health parity

## Architecture & Data
- Architecture / design adjustments:
  - add a small public-lifecycle polling-health reader near `controlServerPublicLifecycle.ts`
  - add a shared operator dashboard presenter near the control observability helpers that reads the compatibility projection and reuses issue payloads
  - historical design note: this slice remapped `uiDataController.ts` onto the operator dashboard presenter; `CO-88` later removed the dead selected-run presenter seam
  - update `packages/orchestrator-status-ui` to render the richer operator dashboard payload, ideally with payload-mode discrimination if preserving the legacy monitor behavior is cheaper than a full replacement
  - keep `/api/v1/refresh` as the operational trigger path instead of introducing a UI-only refresh endpoint
- Data model changes / migrations:
  - additive control observability payload fields only; no persisted migrations expected
- External dependencies / integrations:
  - current compatibility projection and issue payload helpers
  - current control-host public lifecycle refresh coordinator
  - current status-ui static asset package
