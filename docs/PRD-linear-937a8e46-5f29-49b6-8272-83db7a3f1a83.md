## Summary
- Problem Statement: the control host can discover already-active provider-worker runs, but the ordering seam can let capacity gating suppress new dispatch before those already-active discovered runs are attached to provider-intake and projected into `co-status` / `CO STATUS`. At full occupancy, operators can then see a dispatch/admission stall without the already-running provider work being truthfully attached and visible.
- Desired Outcome: already-active discovered provider-worker runs are attached and projected before capacity gating suppresses fresh dispatch, so full-occupancy states remain truthful without changing `max_concurrent_agents`, relying on manual worker starts, or weakening active-run truth validation.
