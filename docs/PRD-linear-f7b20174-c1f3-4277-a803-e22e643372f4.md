## Summary
- Problem Statement: the `CO-58` follow-up review found a remaining evaluation-harness boundary leak. When a checkout uses an external gitdir layout such as `git clone --separate-git-dir <gitdir>`, `evaluation/harness/env.ts` can currently classify the external gitdir parent as the repo root even when that path is not an ancestor of the checkout. That lets `collectNodeModulePaths()` walk above the checkout root and append unrelated ancestor `node_modules` entries to `NODE_PATH`.
- Desired Outcome: add a deterministic reproducer for the external gitdir boundary leak, scope repo-root detection back to the current checkout root for non-ancestor external gitdir layouts, and preserve the nested worktree, symlinked path, and submodule boundaries already repaired in `CO-58`.
