## Summary
- Problem Statement: `orchestrator/src/cli/providerLinearChildLaneShell.ts` still owns its own trailing JSON-tail parser even though `CO-50` already extracted the shared helper into `orchestrator/src/cli/utils/trailingJsonObject.ts`. The local child-lane copy is looser than the shared helper because it does not require the trimmed output to end with `}` before attempting suffix parsing.
- Desired Outcome: child-lane parsing reuses the shared helper from `CO-50`, keeps the same returned child-run payload and path-confinement behavior, and gains focused regression coverage for prelude-log success plus malformed-final-payload failure.
