## Summary
- Problem Statement: same-issue child lanes materialize ephemeral lane workspaces under `.child-lanes/<stream>-<run>`, and the child-lane runner currently inherits the parent `CODEX_HOME`. That leaves the issue-scope seam where Codex trust bookkeeping can treat ephemeral `child-lane paths` as durable trusted projects, causing `trusted-project pollution` and `global ~/.codex/config.toml growth` outside the repo-owned task boundary.
- Desired Outcome: child-lane launches no longer persist ephemeral lane paths into global trusted-project state, while the existing parent-owned child-lane model, explicit acceptance flow, and trust boundaries remain intact and strict.
