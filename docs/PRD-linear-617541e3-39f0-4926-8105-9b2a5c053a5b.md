## Summary
- Problem Statement: provider-worker attempts currently rely on Linear issue state, the `## Codex Workpad`, manifests, and PR/review evidence for continuity, while Codex goals are now stable on Codex CLI `0.135.0+` but not yet deterministically derived from Linear specs or rendered in CO status.
- Desired Outcome: after docs-first/spec creation, each Linear issue attempt creates or reuses exactly one active Codex goal derived from the canonical Linear specs/workpad/checksum, records compact `goal_evidence` as `advisory_only`, and renders `co-status --format json` goal state without making goals lifecycle authority.

## User Request Translation
- User intent / needs: adopt provider-worker goals as a planning and operator-continuity surface for Linear issue attempts. The system should derive one goal objective from the canonical issue packet and active workpad, avoid duplicate overlapping goals, surface missing or stale evidence clearly, and preserve Linear/workpad/review authority.
- Success criteria / acceptance:
  - goal creation happens only after the Linear specs, checksum, and active `## Codex Workpad` exist.
  - the same issue/spec checksum reuses or updates the existing goal instead of creating duplicates.
  - changed checksums supersede old evidence without overlapping active goals.
  - `co-status --format json` renders compact advisory states for missing, unavailable, stale, mismatched-thread, active, and complete evidence.
  - `goal_evidence` retains `advisory_only` authority and never drives Linear lifecycle, PR, review, or closeout decisions.
  - focused tests cover duplicate prevention, objective shaping, status rendering, stale/mismatched evidence, resume, and idempotency.
- Constraints / non-goals:
  - do not replace Linear specs, Linear state, the workpad, PR attachments, review gates, merge closeout, or validation manifests.
  - do not create goals before specs/workpad/checksum are available.
  - do not broaden into release promotion, generic goal UI, or global goal management.
  - do not add extra heavy Linear/GitHub reads when current spec/workpad/checksum evidence already exists.
