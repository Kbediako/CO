# ACTION_PLAN - CO-510 recognize common clean review wording in semantic telemetry

## Summary
- Goal: clear the review-wrapper parser gap that forces waivers for clean Codex review prose.
- Scope: semantic verdict parser, focused tests, review telemetry fixtures, and handoff-gate preservation.
- Non-scope: Linear/PR lifecycle, ready-review, merge, CO-492 goal evidence, broad review-wrapper rewrites, or generic `unknown` relaxation.

## Issue Readiness Gate
- Intent checksum / protected terms: `review/telemetry.json`, `review_verdict`, `clean`, `unknown`, `findings`, `finding_count`, `highest_finding_priority`, `review_outcome: clean-success`, semantic review parser, provider-worker review handoff gate.
- Not done if:
  - every `unknown` verdict becomes clean.
  - `review_outcome: clean-success` alone overrides parsed findings.
  - clean wording masks P-prioritized or structured findings.
  - provider-worker handoff accepts true unknown without a waiver.
  - implementation broadens into lifecycle authority.
- Pre-implementation issue-quality review: this packet confirms CO-510 is an implementation lane with named observed phrases, parser behavior, and negative cases.

## Milestones
1. Packet and registry mirrors.
   - Create PRD, TECH_SPEC, ACTION_PLAN, task checklist, `.agent/task` mirror, task index row, docs/TASKS snapshot, and docs freshness registry rows.
2. Parser discovery.
   - Locate the semantic review verdict parser and existing clean/finding phrase tests.
   - Identify current findings precedence logic before adding clean phrases.
3. Parser update.
   - Add narrowly scoped clean wording recognition for the CO-492 phrases.
   - Preserve findings-first semantics.
   - Preserve true-unknown fail-closed semantics.
4. Tests.
   - Add positive clean phrase cases.
   - Add clean-plus-finding negative cases.
   - Add current-head PR review regressions for no-op actionable-defect labels, validation-only notes, comma-separated defect labels, and benign follow-up prose.
   - Add true-unknown regression coverage if missing.
5. Validation and handoff.
   - Run focused tests, then provider-worker validation floor.
   - Run standalone review and elegance pass before PR handoff.

## Risks & Mitigations
- Risk: overly broad regex turns ambiguous text into clean.
  - Mitigation: restrict patterns to explicit no-defect/no-regression/no-actionable-defect wording and keep negative tests.
- Risk: clean phrase appears in a review that also contains findings.
  - Mitigation: findings precedence remains mandatory.
- Risk: worker broadens into lifecycle gates.
  - Mitigation: keep changes in parser/tests unless existing handoff tests need fixture updates for the same semantic verdict.
- Risk: sentence splitting masks a later concrete defect sentence.
  - Mitigation: allow terminal split clean candidates only after clean/benign prefaces, and constrain prefixed clean subjects with unsafe follow-up and same-line lead-in negative tests.
