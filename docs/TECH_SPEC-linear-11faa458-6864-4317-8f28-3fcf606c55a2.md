## Summary
- Objective: make historical shared repo-local git identity cleanup explicit and operator-safe without reopening CO-120 future leak prevention.
- Scope:
  - docs-first packet registration for `CO-148`
  - one shared-checkout helper for deterministic detection and opt-in cleanup of repo-local `user.name` / `user.email`
  - one bounded update to preventive helper messaging so it points historical cleanup to the shared-checkout path
  - focused regression coverage and exact validation commands
- Constraints:
  - no automatic shared `user.*` mutation in linked-worktree helper flows
  - remediation must run intentionally from the shared checkout root
  - do not rewrite history or change global git config

## Technical Requirements
- Functional requirements:
  - the repo must provide a deterministic shared-checkout detection path for repo-local `user.name` / `user.email` in the shared checkout config file at `/Users/kbediako/Code/CO/.git/config`
  - the repo must provide an explicit, opt-in remediation path that clears those shared repo-local overrides from the shared checkout
  - the remediation helper must fail closed when invoked from a linked worktree or other non-shared-root path
  - the preventive linked-worktree helper must continue to avoid automatic shared `user.*` mutation
  - helper and docs text must distinguish `future leak prevention` from `historical cleanup`
- Non-functional requirements:
  - keep the implementation minimal and easy to audit
  - use machine-checkable output that names the resolved shared config path and present values
  - preserve unrelated shared config, including `extensions.worktreeConfig`, during cleanup
- Interfaces / contracts:
  - historical cleanup seam: `scripts/shared-repo-git-identity.mjs`
  - preventive seam: `scripts/worktree-git-identity.mjs`
  - regression seam: `tests/shared-repo-git-identity.spec.ts` plus existing `tests/worktree-git-identity.spec.ts`

## Architecture & Data
- Architecture / design adjustments:
  - add a separate shared-checkout helper instead of extending the worktree helper with automatic cleanup behavior
  - require the operator to name the shared checkout root explicitly and resolve the shared config path from there
  - keep the worktree helper preventative only, with usage text that redirects historical cleanup to the new helper
- Data model changes / migrations:
  - none
- External dependencies / integrations:
  - local `git`
  - Vitest
