## Summary
- Objective: make the autonomous provider-worker contract require a standalone review plus paired elegance/minimality pass for non-trivial diffs before review handoff, with explicit skip-justification and manual fallback recording rules.
- Scope:
  - docs-first registration and workpad bootstrap for the current Linear worker issue
  - bounded prompt/runtime updates in `providerLinearWorkerRunner.ts`
  - focused prompt/lifecycle regressions in provider-worker tests
  - minimal docs alignment needed to keep the review gate and workpad contract explicit
- Constraints:
  - preserve the existing PR feedback sweep, validation, `origin/main` merge, and PR-check requirements
  - preserve the current review-state routing and merge/rework semantics
  - record delegation override explicitly because this worker run cannot spawn subagents

## Technical Requirements
- Functional requirements:
  - the first-turn provider-worker prompt must require a standalone review before PR create/update intended for review handoff for non-trivial diffs
  - the first-turn provider-worker prompt must require a paired elegance/minimality pass after standalone-review findings are addressed and before review handoff
  - the continuation provider-worker prompt must restate the same non-trivial review-gate contract
  - the prompt must define the default non-trivial heuristic as about `2+ files` or about `40+ changed lines`, unless the worker records an explicit skip justification
  - the prompt must require workpad recording of review goal, findings or fallback, and final clean or justified status
  - the prompt must define a manual fallback path when review tooling is unavailable
- Non-functional requirements (performance, reliability, security):
  - keep the implementation bounded to the current provider-worker prompt builder and nearby tests/docs
  - keep the resulting worker contract auditable and copyable from the prompt text itself
  - avoid adding new lifecycle branching or provider features unrelated to the review gate
- Interfaces / contracts:
  - provider-worker prompt contract: `orchestrator/src/cli/providerLinearWorkerRunner.ts`
  - provider-worker prompt/lifecycle tests: `orchestrator/tests/ProviderLinearWorkerRunner.test.ts`
  - workflow facade regression coverage: `orchestrator/tests/ProviderLinearWorkflowFacade.test.ts`
  - review policy baseline: `AGENTS.md`, `skills/standalone-review/SKILL.md`, `skills/elegance-review/SKILL.md`, `skills/linear/SKILL.md`, `docs/standalone-review-guide.md`
