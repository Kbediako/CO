# PRD - CO Recalibrate Diff Budget for Stacked Lanes While Keeping Hard PR Scope Guard

## Summary
- Problem Statement: CO still benefits from a hard diff-budget guard, but the current local working-tree implementation measures against `origin/main` aggregate scope and uses the same hard budget that CI uses for explicit PR-base scope. On stacked lanes, that makes the local signal low-yield and override-heavy because the gate answers the broad branch-history question rather than the current reviewable lane question.
- Desired Outcome: local implementation-gate diff-budget behavior becomes truthful for stacked lanes by hard-gating the current reviewable head scope, broad aggregate stacked scope is still surfaced as advisory evidence rather than hidden, CI and explicit PR/base scope remain hard-gated, and the default line budget aligns with the already-documented review large-scope policy plus current docs-first mirror expectations.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): Complete Linear issue `CO-15` in this workspace by recalibrating diff-budget so stacked CO lanes stop needing routine local overrides while preserving a hard PR/current-head guard and a clear audit trail.
- Success criteria / acceptance:
  - local implementation-gate diff-budget behavior is truthful on stacked lanes
  - CI and explicit PR/base scope remain hard-gated
  - thresholds and docs-first accounting align with the current review large-scope policy
  - routine stacked-lane override usage drops materially from the current baseline
- Constraints / non-goals:
  - do not remove the hard diff-budget concept
  - do not weaken CI PR gating
  - keep the change narrow to diff-budget behavior, the explicit workflow contract, tests, and required docs
  - delegation must be recorded as an explicit override in this worker run because subagent spawning is unavailable in-session

## Goals
- Make default local working-tree diff-budget runs evaluate current-head scope as the hard pass/fail signal.
- Preserve explicit base and commit scopes as hard-gated surfaces for PR and CI usage.
- Surface stacked `origin/main` aggregate scope as explicit advisory evidence when it is broader than the current local lane.
- Align the default line threshold with the existing standalone-review large-scope policy.
- Lock the new behavior with focused script and workflow-facing regressions.

## Non-Goals
- Rewriting unrelated review-runtime behavior.
- Removing aggregate scope visibility entirely.
- Changing the Linear workflow beyond the normal provider-worker requirements for this issue.
