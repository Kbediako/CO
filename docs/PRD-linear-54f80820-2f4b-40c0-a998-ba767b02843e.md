# PRD - CO-513 suppress same-attempt retries for create-follow-up label-resolution failures

## Summary
- Problem Statement: `linear_follow_up_label_resolution_failed` is fail-closed and non-retryable, but same-attempt provider-worker retry suppression does not classify it yet.
- Desired Outcome: a provider worker that hits source-label resolution failure receives deterministic no-retry guidance to fix/source-label the source issue first.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): close the CO-482 sidecar gap where `create-follow-up` can be retried against the same under-labeled source issue during one provider-worker attempt.
- Success criteria / acceptance: classify the label-resolution failure as deterministic, emit a clear no-retry instruction, and cover it with a focused regression.
- Constraints / non-goals: preserve CO-482 fail-closed label assignment; do not broaden into CO-400 label projection, cached label authority, or generic label taxonomy changes.

## Intent Checksum
- Exact user wording / phrases to preserve: `linear_follow_up_label_resolution_failed`, `same-attempt retries`, `create-follow-up`, `source-label resolution failures`, `fail-closed`, `non-retryable`, `CO-482`, `CO-400`.
- Protected terms / exact artifact and surface names: `orchestrator/src/cli/control/providerLinearWorkerTruth.ts`, `orchestrator/src/cli/linearCliShell.ts`, `resolveFollowUpLabelsFromSourceIssue`, `linear_follow_up_label_resolution_retry_suppressed`.
- Nearby wrong interpretations to reject: do not make label resolution retryable; do not silently create under-labeled follow-ups; do not project missing labels from cache or stale issue data.

## Parity / Alignment Matrix
- Not applicable. This is a retry-classification and retry-suppression guidance lane, not a parity/alignment lane.

## Not Done If
- The worker can keep retrying `create-follow-up` in the same attempt after a source-label resolution failure.
- The fix broadens into CO-400 label projection or cached label authority.
- The fix weakens CO-482 label failure semantics.

## Goals
- Classify `linear_follow_up_label_resolution_failed` as deterministic provider mutation failure.
- Suppress same-attempt `create-follow-up` retry with source-label repair guidance.
- Add a focused regression for the no-retry instruction.

## Non-Goals
- Linear label taxonomy changes.
- Generic label projection or cached label authority work.
- Changes to CO-482 live source-derived label assignment requirements.

- Dependencies / Integrations: provider Linear audit entries and live source-label resolution from CO-482.

- Trigger: same-attempt retry after `create-follow-up` recorded `linear_follow_up_label_resolution_failed`.
- Introduced date: 2026-05-14.
