# TECH_SPEC Mirror - CO-571 released terminal claim reconciliation restart loop

Canonical spec: `tasks/specs/linear-cdf8b078-95af-4e75-99e2-6c32fa1ecd63.md`; PRD: `docs/PRD-linear-cdf8b078-95af-4e75-99e2-6c32fa1ecd63.md`; action plan: `docs/ACTION_PLAN-linear-cdf8b078-95af-4e75-99e2-6c32fa1ecd63.md`.

## Contract
CO-571 fixes provider refresh lifecycle classification for retained released terminal historical claims, parked Backlog released `not_active` claims, and stale retained merged-closeout history. `claim_issue_by_id:released` and `claim_reconcile:released` records for terminal Done or Duplicate/canceled issues with retry fields null and no active worker/live issue corroboration remain audit-visible but must not drive `provider_refresh_lifecycle_stuck` / `restart_required` when WIP is `0/3` or when a selected released claim is inconsistently projected as retrying. This includes both `run_id=null` claims and non-cancellable historical run_id/run_manifest_path bindings. PR #856 extends the contract to the no-current-poll-snapshot path: a strong terminal released row must not fall through to direct issue-by-id only because `resolveTrackedIssues` skipped or was unavailable, while accepted `provider_issue_rehydration_pending_revalidation` claims must still revalidate. The second rework extends the contract to current-poll terminal snapshots: when the poll map reconfirms equal-or-newer terminal `provider_issue_released:not_active` truth, the row must remain passive before `claim_reconcile:released` progress or claim churn; active/reopened current-poll snapshots must still relaunch. The third rework extends the same passive classification to map-missing terminal snapshots: completed/canceled issue filtering can omit a terminal issue such as CO-480 from the current poll map, and that omission must not make strong cached terminal `not_active` history active. The fourth rework extends the direct issue-by-id guard to CO-529-style Backlog/backlog released `not_active` rows with no active run, retry, promotion, merge-closeout, or cancelable retained-run evidence. The sixth rework initially extended that Backlog guard to the CO-521/CO-524 no-current-poll-snapshot recurrence; Codex review tightened the contract so no-current-poll Backlog rows are hidden only after a bounded direct issue-by-id read verifies the same inert Backlog snapshot and records explicit recent `passive_release` metadata, while active direct truth still starts work. The fifth rework extends the guard to CO-522-style stale retained `review_promotion` / `merge_closeout` history after PR #795 merged and live Linear truth is Done/completed, while preserving current promotion, active/reopened, retry, and direct live Done normalization paths. Retained `review_promotion` metadata is stale only when newer terminal issue truth supersedes it; current promotion metadata must still revalidate, including deferred-poll fail-closed paths. Real active refresh stalls must still fail closed with provider keys and phase evidence.
