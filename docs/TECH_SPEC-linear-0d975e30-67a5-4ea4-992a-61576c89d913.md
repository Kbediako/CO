## Summary
CO-524 adds a structured `review_intent_matrix` so review prompt assembly, review telemetry interpretation, and `provider-worker handoff` use the same intent source map. The matrix must expose required axes for `original_spec`, `coding_standards`, `code_change_proposals`, and `agent_loop_change_proposals`; the `original_spec` axis must prove canonical `PRD`, `TECH_SPEC`, and `ACTION_PLAN` evidence reached the reviewer.

## Technical Requirements
- Matrix axes: include `required`, `verdict`, `evidence`, and `proposed_fixes`.
- Required axes: `original_spec`, `coding_standards`, `code_change_proposals`, and `agent_loop_change_proposals`.
- Required `original_spec` evidence: canonical `PRD`, `TECH_SPEC`, and `ACTION_PLAN` availability or explicit unavailable evidence.
- Telemetry boundary: `review_verdict` findings, unknown, missing, stale, or mismatched evidence blocks clean handoff even when `review_outcome` is `clean-success` or `bounded-success`.
- Wrapper boundary: failed or bounded wrapper outcomes remain visible through `review_outcome` and cannot erase semantic findings.
- Compatibility: Codex CLI 0.130.0 evidence must be explicit, fixture-backed, and fail closed when matrix metadata is missing or stale.
- Authority boundary: `goals advisory-only` context cannot satisfy or override review intent, review verdict, review outcome, or provider-worker handoff gates.
