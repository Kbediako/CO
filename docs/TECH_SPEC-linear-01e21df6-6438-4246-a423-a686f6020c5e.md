## Summary
- Objective: replace raw reward-only top-K experience retrieval with prompt-pack-declared competitive scoring and anti-dominance normalization while keeping prompt injection bounded and backward-compatible.
- Scope:
  - extend prompt-pack manifests and loader output with a controller-consumable retrieval policy
  - extend the Experience Store retrieval path to compute competitive scores, dominance penalties, and bounded diagnostics
  - extend manifest `prompt_packs` entries to expose retrieval policy plus selection metadata and a diagnostics artifact path
  - add focused tests for repeated-source suppression, ranking changes, prompt-pack parsing, and manifest compatibility
- Constraints:
  - no TF-GRPO rewarder rewrite
  - no memory-backend expansion
  - no telemetry/runtime continuity/parity expansion

## Implementation Boundary
- Controller surface:
  - add prompt-pack retrieval policy fields in `.agent/prompts/prompt-packs/**/manifest.json`
  - parse and validate them in `packages/orchestrator/src/instructions/promptPacks.ts`
- Retrieval engine:
  - keep storage in `orchestrator/src/persistence/ExperienceStore.ts`
  - add one explicit competitive-selection path that returns both records and diagnostics
- Manifest surface:
  - add manifest-visible retrieval policy and selection summary fields under `prompt_packs[]`
  - keep the existing `experiences` string array for prompt consumers
- Prompt consumers:
  - no structural prompt changes required beyond optionally richer snippet text
  - cloud and other prompt readers continue to use `experiences`

## Design
- Prompt-pack retrieval policy:
  - versioned additive object on each prompt-pack manifest
  - weighted score terms for `gtScore` and `relativeRank`
  - anti-dominance normalization parameters including enable flag, provenance grouping, and strength
  - optional `min_score` threshold for gating weak candidates
- Experience provenance grouping:
  - derive a stable repeated-source key from stored provenance in priority order: `groupId`, then `runId`, then `manifestPath`, then `stampSignature`
  - persist that grouping decision in retrieval diagnostics so operators can see which provenance surface was used
- Competitive scoring:
  - compute a raw score per candidate from configured weights
  - greedily fill slots; at each slot recompute each remaining candidate’s `competitive_score = raw_score - dominance_penalty`
  - dominance penalty scales with prior selections from the same provenance source and the configured anti-dominance strength
  - deterministic ties break by newer `createdAt`, then record `id`
- Diagnostics:
  - write one per-pack retrieval JSON artifact under the run artifact root
  - include policy, source grouping, candidate scores, dominance penalties, selected slot order, and exclusion reasons
  - surface the artifact path plus selected IDs and suppressed source keys in the manifest
