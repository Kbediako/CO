## Summary
- Problem Statement: `provider merge closeout` can finish the `provider closeout lifecycle` by moving an issue to `Done` after a `metadata PR merge` even when `origin/main` still configures that issue as the `docs:freshness:maintain` `owner_issue`. The CO-444 incident showed the failure mode: terminal owner state later makes maintenance fail closed with `configured_owner_terminal`, `block_unowned_repo_debt`, and `blocking_changed_paths=[]`.
- Desired Outcome: `providerMergeCloseout.ts` checks live `docs:freshness:maintain` owner status after merge/shared-root reconciliation and before terminal transition. If the closing issue is still the live owner, closeout records machine-readable owner evidence and leaves or moves the issue to a non-terminal owner-bearing state, likely `Blocked`. If ownership has been re-homed or retired, normal terminal closeout still works.

- User intent / needs: fix the provider closeout lifecycle bug rooted from CO-444 without re-homing ownership as the final answer and without weakening docs freshness.
