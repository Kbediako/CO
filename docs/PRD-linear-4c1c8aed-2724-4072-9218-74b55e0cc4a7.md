## User Request Translation

Create the docs-first packet for `CO-264` only. The issue asks for a retention policy for `backlog_promotion_snapshots` so provider autopilot does not discard useful backlog-promotion context just because a tracked issue temporarily disappears from one finite tracked-issue page.

The selected policy is retention-first: retain snapshots through temporary untracked cycles by default, and prune only when explicit stale or permanent evidence exists. Preferred pruning evidence is terminal workflow state recorded on the snapshot/read model. If terminal-state evidence is unavailable, an explicit bounded stale counter may be used, but it must not fire on a single missing tracked page.

## Problem Statement

`providerOperatorAutopilot.ts` can only make safe backlog-promotion decisions when it has enough evidence about the queue head, prior promotion context, and whether an issue has become permanently stale. A single missing page from finite tracked-issue paging is not terminal evidence. If the system treats one untracked cycle as deletion proof, it can drop `backlog_promotion_snapshots`, lose auditability, and potentially reopen the `CO-216` manual-demotion suppression behavior without an explicit retention policy and regressions.

This is a docs/spec packet, not an implementation lane. The parent implementation should make retention and pruning decisions observable through `providerWorkflowConfigStore.ts` and `observabilityReadModel.ts` after this policy is accepted.

## Desired Outcome

- `backlog_promotion_snapshots` survive temporary untracked cycles by default.
- `resolveNextBacklogPromotionSnapshots` selects, retains, or prunes snapshots from explicit decision inputs instead of blanket age-only deletion.
- A single missing tracked page is never treated as terminal evidence.
- Pruning requires explicit stale/permanent evidence, preferably terminal workflow states recorded on the snapshot/read model.
- Any bounded stale counter is explicit, audited, and requires more than one missing cycle.
- Audit/read-model output records age, cycle count, terminal evidence, tracked-page visibility, and force-path status for each retention/prune decision.
- `CO-216` manual-demotion suppression remains intact until this retention policy and focused regressions exist.
