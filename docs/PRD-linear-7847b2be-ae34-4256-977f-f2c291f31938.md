## Summary
- Problem Statement: `co-status --format json` can report current failed work from the outer provider manifest when the worker handoff actually reached terminal success. The protected shape is a `provider manifest` / `outer provider manifest` status of failed with `Run provider linear worker` exit code `1`, while `worker proof` says `owner_phase=ended`, `owner_status=succeeded`, the PR is merged, Linear terminal closeout is `Done`, `retrying=0`, and no retry metadata exists.
- Desired Outcome: define the CO-563 docs-first packet and implementation contract so `co-status --format json` uses a clear authority order across worker proof, provider manifest status, PR/Linear terminal state, and retry state. The target behavior reconciles terminal-success handoffs without hiding wrapper diagnostic metadata or real active failed workers.
