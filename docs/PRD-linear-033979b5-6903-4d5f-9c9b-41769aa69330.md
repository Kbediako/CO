## Summary
- Problem Statement: `ready-review` currently has a rereview request classifier that can treat a human acknowledgement-only `@coderabbitai` mention as a fresh CodeRabbit rereview or resolve request. That can leave a PR blocked on `bot_rereview_pending=coderabbitai` even when the comment was only an acknowledgement of already-handled feedback.
- Desired Outcome: `ready-review` distinguishes acknowledgement-only CodeRabbit mentions from explicit rereview or resolve requests. Acknowledgement-only comments do not create or extend a CodeRabbit rereview wait; explicit requests still block until completed; existing gates for `unacknowledged_bot_feedback`, unresolved review threads, CodeRabbit check state, and required checks remain unchanged.

## User Request Translation
- User intent / needs: Create the docs-first packet for a narrow parent implementation lane that fixes CodeRabbit rereview request intent classification inside `ready-review` without loosening any review-readiness guard.
- Success criteria / acceptance:
  - `@coderabbitai` acknowledgement-only comments such as addressed/fixed/thanks-style replies do not become latest CodeRabbit rereview requests.
  - Explicit `@coderabbitai` rereview, re-review, review again, or resolve requests still become latest CodeRabbit requests and keep `ready-review` blocked until a valid completion signal exists.
  - Existing `unacknowledged_bot_feedback` behavior is preserved: current-head actionable bot comments without human replies still block.
  - Existing unresolved review-thread behavior is preserved: non-outdated unresolved threads still block.
  - Existing CodeRabbit status/check behavior is preserved: missing, pending, failed, skipped, stale, or older-than-request CodeRabbit evidence does not clear an explicit request.
  - Focused regression coverage pins acknowledgement-only, explicit rereview, explicit resolve, and existing gate preservation cases.
- Constraints / non-goals:
  - Do not edit source or tests in this docs-packet child lane.
  - Do not mutate Linear, workpad, GitHub, PR, or review lifecycle state from this child lane.
  - Do not weaken `unacknowledged_bot_feedback`.
  - Do not weaken unresolved review-thread handling.
  - Do not weaken CodeRabbit checks or explicit rereview request handling.
  - Do not run full repo validation suites from this child lane.
