# PRD - CO: Make provider-worker closeout reporting authoritative and suppress deterministic invalid retries

## Added by Bootstrap 2026-04-04

## Summary
- Problem Statement: `CO-78` exposed a provider-worker truthfulness gap. A lane can fail terminally while closeout surfaces still contain success-shaped text, workpad/review/summary artifacts can drift from the real terminal result, deterministic invalid follow-up mutations can be retried as though they were transient, and auxiliary proof harness manifests can continue to look like active provider work.
- Desired Outcome: the provider-worker closeout path becomes authoritative. Failed or degraded lanes read as failed everywhere that matters, deterministic invalid provider mutations are classified once and suppressed for the rest of the attempt, and proof-harness sidecars are finalized or excluded so they do not pollute operator-facing runtime truth.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): complete `CO-79` as a bounded truthfulness pass over provider-worker closeout/reporting so a partially completed but terminally failed autonomy run cannot look clean or half-successful in any primary artifact, while keeping the fix narrow and regression-backed.
- Success criteria / acceptance:
  - manifest, run-summary, out/state, and workpad closeout all derive from the same authoritative terminal result
  - authoritative review telemetry is reflected truthfully in closeout text, especially for `bounded-success`, `failed-boundary`, and generic failure cases
  - deterministic invalid provider mutations such as parity-matrix-missing follow-up creation are not retried blindly in the same attempt
  - auxiliary proof harness runs are finalized cleanly or excluded from active provider truth surfaces
  - focused regression coverage reproduces the `CO-78` class: partial progress, deterministic invalid mutation, auxiliary proof artifacts, and terminal failure truth
- Constraints / non-goals:
  - no historical artifact rewriting for older tasks
  - no replacement of the workpad model itself
  - no general report beautification unrelated to failure truth
  - keep the fix bounded to provider-worker closeout, mutation classification, and truth surfaces

## Parity / Alignment Matrix
- Current truth:
  - closeout/reporting text can drift from the authoritative terminal manifest result
  - review telemetry classification can be flattened into success-shaped or ambiguous closeout language
  - deterministic invalid provider writes can be retried within the same attempt
  - auxiliary proof manifests can remain `in_progress` and appear as live provider work
- Reference truth:
  - provider-worker closeout should derive from one authoritative terminal result plus authoritative review telemetry
  - deterministic invalid provider mutations should be fail-closed and non-retryable within an attempt
  - sidecar proof artifacts should either end terminally or be excluded from active operator truth surfaces
- Target truth / intended delta:
  - every primary closeout surface agrees on terminal success, degradation, or failure
  - deterministic invalid follow-up creation is classified once and suppressed for the remainder of the attempt
  - proof sidecars are either finalized or quarantined from running/runtime truth
- Explicitly out-of-scope differences:
  - backfilling old task artifacts
  - redesigning the workpad format or review wrapper model
  - unrelated provider-worker UX polish

## Not Done If
- A terminally failed provider-worker lane can still look clean or completed in any primary closeout surface.
- Deterministic invalid follow-up mutations are still retried within the same attempt.
- Auxiliary proof manifests can still appear as active provider work in operator-facing truth surfaces.
- Review telemetry outcomes are collapsed into misleading success/failure wording.

## Goals
- Make provider-worker terminal closeout derive from authoritative manifest and review telemetry truth.
- Classify deterministic invalid provider mutations once per attempt and suppress blind retries.
- Prevent auxiliary proof harness runs from polluting active provider-worker truth.
- Add focused regression coverage for the mixed partial-success / terminal-failure lane class.

## Non-Goals
- Rewriting historical manifests, summaries, or workpads for already-closed tasks.
- Replacing the Linear workpad mechanism or its structural validation rules.
- Generalized report copy refresh unrelated to truthfulness.
