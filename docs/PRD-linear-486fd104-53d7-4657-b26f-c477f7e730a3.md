# PRD - CO: Make provider-worker closeout reporting authoritative and suppress deterministic invalid retries

## Traceability
- Linear issue: `CO-79` / `486fd104-53d7-4657-b26f-c477f7e730a3`
- Linear URL: https://linear.app/asabeko/issue/CO-79/co-make-provider-worker-closeout-reporting-authoritative-and-suppress
- Source issue: `CO-78`

## Problem
- `CO-78` showed provider-worker truth drift: a lane could fail terminally while manifest-backed closeout or status surfaces still sounded successful.
- Deterministic invalid follow-up creation such as `linear_follow_up_parity_matrix_missing` could be retried inside the same attempt.
- Auxiliary proof harness manifests could stay visible as active provider work.

## Outcome
- Primary closeout surfaces derive from one authoritative terminal result plus fresh review telemetry.
- Deterministic invalid provider mutations are classified once and suppressed for the rest of the attempt.
- Auxiliary proof harness runs no longer pollute operator-facing active truth.
- Focused regressions cover the mixed partial-success and terminal-failure `CO-78` class.

## Protected Expectations
- A failed provider-worker lane must read as failed everywhere that matters.
- Deterministic invalid mutations are not transient noise and should not be retried blindly.
- Auxiliary proof runs must not pollute operator-facing run truth.
- `bounded-success`, `failed-boundary`, and `failed-other` review outcomes must stay explicit.

## Non-Goals
- Rewriting historical artifacts for older tasks.
- Replacing the workpad model.
- General report beautification unrelated to truthfulness.
