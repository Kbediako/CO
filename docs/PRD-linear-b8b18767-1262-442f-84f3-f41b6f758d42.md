# PRD - CO-533 docs/TASKS archive reserve restoration

## Summary
- Problem Statement: `docs/TASKS.md` can sit inside the configured reserve window while the supported archive command cannot restore the 440-line target. Ordinary packet PRs then inherit review churn and one-off archive cleanup.
- Desired Outcome: `npm run docs:archive-tasks` restores the configured reserve target on current main without weakening `docs:check`, deleting historical task truth, or asking unrelated packet lanes to fix shared headroom.

## User Request Translation (Context Anchor)
- User intent / needs: fix the root cause behind recurring docs headroom failures instead of squeezing or removing one new packet row.
- Success criteria / acceptance: reproduce the failing reserve path, identify the stale metadata and archive-index parsing causes, land durable parser/diagnostic coverage, restore the reserve, and preserve archive truth.
- Constraints / non-goals: do not widen or zero `reserve_lines`, do not remove `tasks-file-too-large`, do not fold CO-530 freshness cohort classification into this lane, and do not mark nonterminal task rows completed without evidence.

## Intent Checksum
- Exact user wording / phrases to preserve: "we shouldn't just patch then over, we should dive deep get to the root cause and fix it".
- Protected terms / exact artifact and surface names: `docs/TASKS.md`, `docs/TASKS-archive-2026.md`, `docs/tasks-archive-policy.json`, `max_lines=450`, `reserve_lines=10`, `target_lines=440`, `npm run docs:archive-tasks -- --dry-run`, `Unable to restore reserve target 440`, `tasks/index.json`.
- Nearby wrong interpretations to reject: removing only the newest packet row, weakening `docs:check`, hiding the archive failure, or blindly completing every terminal-looking historical snapshot.

## Parity / Alignment Matrix
- Current truth: current main starts at 448 lines and the dry-run cannot go below 448/447 because archive-index structure and stale index metadata block eligible completed rows.
- Reference truth: CO-221 established a non-zero reserve target and a proactive archive helper that should restore `docs/TASKS.md` to 440 lines.
- Target truth / intended delta: the helper treats archive-index as structural metadata, emits actionable blocked-metadata diagnostics, and archives verified completed rows until the reserve target is restored.
- Explicitly out-of-scope differences: generic docs freshness cohort caps, Linear queue admission policy, and CO-451 handoff status.

## Not Done If
- `docs/TASKS.md` remains above 440 after the supported archive command.
- Archive output loses displaced task evidence or the archive-index line.
- The command fails with an opaque reserve error when stale index metadata is the blocker.
- The fix relies on a manual one-off row removal.

## Goals
- Restore `docs/TASKS.md` reserve headroom on the current branch.
- Add regression coverage for archive-index adjacency and nonterminal index-status diagnostics.
- Correct only verified completed historical metadata needed for reserve restoration.

## Non-Goals
- No freshness policy relaxation.
- No broad task registry cleanup.
- No changes to provider-worker admission or Linear WIP limits.
