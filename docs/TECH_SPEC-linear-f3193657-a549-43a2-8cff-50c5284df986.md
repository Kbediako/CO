## Architecture & Data
- Architecture / design adjustments:
  - add a pause-handoff guard so an in-flight live refresh cannot immediately append a second paused primary redraw after the handoff snapshot
  - derive authoritative running contexts from the selected run and active provider-intake claims rather than raw historical manifest status alone
  - keep same-issue dedupe rules aligned so running rows, issue payloads, and aggregate telemetry all use the same authoritative representative
  - aggregate token totals with null-aware semantics and carry authoritative rate limits from the freshest live budget or proof source
- Data model changes / migrations:
  - no external schema migration
  - payload semantics change from zero-coercion to null-aware token totals when authority is absent
  - rate-limit source precedence becomes explicit
- External dependencies / integrations:
  - provider intake state for current-claim authority
  - provider worker proof and live Linear budget snapshots for rate limits and token telemetry
  - screenshot proof uploaded into Linear through the workpad refresh flow
