## Technical Considerations
- Architectural Notes:
  - `printControlHostSupervisionStatus()` currently reads `launchctl print`, the stored supervision `state.json`, and the LaunchAgent plist, then renders the payload without a live `co-status --format json` probe
  - `buildControlHostSupervisionStatusPayload()` currently derives `service.loaded` from `launchctl.exitCode === 0`
  - `formatControlHostSupervisionStatus()` currently echoes persisted `state.status` and `state.last_health_status`
  - `runControlHostSupervision()` currently persists `status: 'restart_required'` and `last_health_status: probe.reason` after the unhealthy threshold is hit
  - `probeControlHostHealth()` currently uses `co-status --format json` as the live health probe surface
  - `providerControlHostFreshnessGauge.ts` separately discovers and reads `provider-intake-state.json`
- Dependencies / Integrations:
