## Summary
- Problem Statement: the issue description is specific to local host pressure, not a generic control-host outage. Under concurrent local `provider-worker` load, long-lived workers plus additional lanes can push local Node memory pressure toward `SIGABRT` / `V8 OOM`, while repeated `provider_worker_child_lane_provenance_invalid` failures can create noisy `provenance-invalid retry churn` instead of containing the pressure truthfully.
