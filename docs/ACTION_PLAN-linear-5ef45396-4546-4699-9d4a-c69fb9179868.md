## Risks & Mitigations
- Risk: CO-396 is read as permission to remove all runtime fallback support.
  - Mitigation: packet separates expiring auto reroutes from durable compatibility adapters, fail-fast metadata, audit schema, and validation canary contracts.
- Risk: cloud and runtime fallback semantics blur.
  - Mitigation: packet protects `runtime_fallback`, `cloud_fallback`, required cloud canary, and fallback cloud canary as distinct terms.
- Risk: implementation starts before traceability exists.
  - Mitigation: the implementation now builds on the registered packet and keeps source changes scoped to existing runtime-routing contracts.
