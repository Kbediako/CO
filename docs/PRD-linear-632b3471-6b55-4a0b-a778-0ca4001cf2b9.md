## Summary
- Problem Statement: CO-557 must prevent provider docs-review evidence from drifting across task-key and artifact-root authority. A `docs-review` child stream can be confused with docs packet or source-freshness child keys such as `linear-<uuid>-docs-packet` or `linear-<uuid>-source-freshness-recheck`, while control-host artifact lookup can drift between the shared control-host artifact root and workspace-local `.runs`. That combination can make `delegation-guard`, `linear child-lane --action accept`, accepted child-lane manifest lookup, and `provider-intake-state.json` evidence disagree about the authoritative parent task.
- Desired Outcome: parent implementation makes the provider `docs-review` task-key and control-host artifact root explicit and fail-closed. `linear child-stream --pipeline docs-review` must bind to the registered parent task in `tasks/index.json`, accepted child-lane manifests must be resolved from the correct root, and source-freshness recheck artifacts must remain visible without becoming docs-review authority.
- Current-main validation addendum: while validating the parent implementation, `docs:check` exposed a separate docs/TASKS reserve failure where `scripts/tasks-archive.mjs` did not treat `tasks/index.json` status `done` as a terminal index status. The parent lane may include the narrow root-cause archive fix because lowering the cap, deleting history, or relabeling old rows would only mask the recurring gate failure.
