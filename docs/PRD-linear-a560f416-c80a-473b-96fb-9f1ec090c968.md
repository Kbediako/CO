# PRD - CO-172 Provider Worker Handoff Manifest Truth
Problem/goal/acceptance: `provider-linear-worker` can complete the primary issue outcome successfully, including a successful handoff/merge, while the final manifest summary is later dominated by post-handoff guardrail-summary collection noise such as `Guardrails: spec-guard command not found`. CO-172 preserves the guardrails and their evidence, but changes final manifest status and final manifest summary precedence so primary worker outcome truth is not overwritten by secondary post-handoff cleanup, Linear refresh, or guardrail-summary collection failures. The lane is complete only when recorded CO-169 / PR #465 and CO-171 / PR #466 evidence or an equivalent focused fixture reproduces the ambiguity, regressions prove successful handoff/merge summary precedence, and secondary failures remain machine-checkable instead of silent.

Not done if: a successful handoff/merge can leave the final manifest failed solely because guardrail-summary collection could not find spec-guard; a successful final manifest still has `Guardrails: spec-guard command not found` as the only or leading summary; post-handoff cleanup, Linear refresh, or guardrail-summary failures become silent; or the change only documents a manual waiver without changing the manifest truth contract.
