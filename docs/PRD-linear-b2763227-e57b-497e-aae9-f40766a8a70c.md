## Parity / Alignment Matrix
- Required for parity/alignment lanes; otherwise state `Not applicable`.
- Current truth:
  - `tests/cli-source-bootstrap.spec.ts` waits for `ready\n`, then immediately sends `SIGTERM` to the bootstrap process
  - the fixture source body currently writes `ready\n` before registering `process.once('SIGTERM', ...)`
  - `bin/codex-orchestrator.js` forwards stop signals to the re-execed child and re-emits after the child closes
  - `CO-134` already changed the assertion from a one-shot `readFile(...)` to polling `waitForFileContents(...)`, but the source-body ordering remains unchanged
- Reference truth:
  - the test's readiness signal should mean the source entrypoint is already armed to handle the forwarded signal
  - the regression should fail only when bootstrap signal forwarding or source-entrypoint handling actually regresses
- Target truth / intended delta:
  - the signal-forwarding regression has a deterministic readiness contract
  - `child-signal.txt` `ENOENT` no longer happens because the test fires `SIGTERM` before the fixture can handle it
  - remaining failures on this surface map to real bootstrap regressions or explicitly documented platform assumptions
- Explicitly out-of-scope differences:
  - unrelated CI rerun policy
  - broad provider-worker orchestration refactors
  - new abstractions for bootstrap launch or signal handling unless the current implementation proves insufficient
