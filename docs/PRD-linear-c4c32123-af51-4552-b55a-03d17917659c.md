## Summary
- Problem Statement: `CO-55` made `co-status` easy to launch, and `CO-60` made the foreground live monitor scrollback-clean by moving normal live redraw into the terminal alternate screen. The current gap is a different operator flow: when a JSON `control-host` is already running, there is still no simple read-only terminal command that attaches to that host, and the only shipped `co-status` surface still launches the host path itself. Operators who only want live terminal status from the already-running host must either start another host path or give up live scrollback until they press `p`.
- Desired Outcome: add a dedicated attach/viewer command that resolves an already-running local JSON `control-host` from its persisted endpoint/auth artifacts, fetches the read-only dashboard data from that host, and renders a live `CO STATUS` attach viewer in primary scrollback, so operators can keep scrolling while the viewer remains live. The existing launch path should stay available, but help/docs must make the launch-vs-attach split explicit.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): preserve the current launched `CO STATUS` host experience, but add the missing operator-friendly read-only attach mode for the already-running host. The attach viewer must not guess routes, must not start another poller or host, and must keep live terminal scrollback available without forcing the operator to pause first.
- Success criteria / acceptance:
  - a dedicated attach/viewer command exists and is discoverable in CLI help
  - attach resolves the local host from existing run-dir endpoint/auth artifacts and/or documented explicit task/run/run-dir flags
  - attach is read-only and does not launch another `control-host`, Linear poller, or Telegram poller
  - live attach rendering supports scrollback without pressing `p` first
  - help/docs explain the difference between `co-status` launch and `co-status attach`
  - final validation demonstrates the real attach flow against an already-running local JSON host and embeds terminal screenshots directly in Linear
- Constraints / non-goals:
  - preserve the current `co-status` launch alias for starting the live host-backed dashboard
  - do not replace the browser `/ui` surface
  - do not require a host restart just to inspect status
  - do not turn the attach viewer into a mutating control surface

## Not Done If
- `co-status attach` still launches a new host or relies on undocumented internal routes.
- The attach viewer still uses live alternate-screen semantics that require `p` before scrollback is useful.
- The attach flow cannot resolve the host from persisted run-dir artifacts or clear explicit flags.
- Help/docs do not explain launch vs attach clearly.
- Final validation lacks real local-host attach proof embedded directly in Linear.

## Goals
- Add a dedicated read-only attach command for an already-running local JSON `control-host`.
- Reuse the host's authenticated read-only dashboard data surface instead of starting another control runtime.
- Make the attach viewer live-scrollback-friendly while keeping useful local inspect affordances.
- Document and test the attach contract explicitly.

## Non-Goals
- Replacing or widening the browser dashboard.
- Changing provider polling cadence or lifecycle ownership in the running host.
- Removing the current launched `co-status` flow or its paused inspect behavior.
- Adding remote/non-local attach support beyond the persisted local host artifacts in this lane.

## Technical Considerations
- Architectural Notes:
  - `bin/codex-orchestrator.ts` currently routes `co-status` directly into the `control-host` launch path
  - the running host already exposes authenticated read-only dashboard data at `/ui/data.json`
  - the host run dir already persists `control_endpoint.json` and `control_auth.json` sidecars that can identify the local host without route guessing
  - the narrow target is:
    - add `co-status attach` as an explicit CLI subcommand
    - resolve the host from run-dir artifacts (`task` + `run` defaults or explicit `run-dir`)
    - render a read-only viewer locally from fetched dashboard data
    - keep the attach viewer on primary scrollback while live, so scrolling remains available
  - the attach viewer must not invoke mutating host actions; at most it may read the dashboard dataset and apply local-only inspect controls
  - launch-vs-attach parity matrix:

    | Concern | Current shipped launch path | Reference contract to preserve | Target attach path |
    | --- | --- | --- | --- |
    | Command surface | `bin/codex-orchestrator.ts` routes `co-status` into the launched host-backed viewer path. | `co-status` remains the explicit launch command for starting and owning the local host session. | `bin/codex-orchestrator.ts` adds `co-status attach` as the explicit read-only viewer command for an already-running host. |
    | Host ownership | Launch mode owns host lifecycle and can create or resume the local `control-host` run before rendering status. | The launched path keeps host ownership and remains the only terminal surface that starts the host path itself. | Attach mode resolves `control_endpoint.json` and `control_auth.json` from the existing run dir or explicit flags through the shared control-endpoint loader in `orchestrator/src/cli/delegationServer.ts`; it never starts a second host. |
    | Screen mode | Live launched `CO STATUS` renders through `controlStatusDashboard.ts` on the terminal alternate screen when the effective output is interactive. | Preserve the launched alternate-screen ergonomics so existing operators do not lose the current live dashboard behavior. | Attach mode reuses `controlStatusDashboard.ts` with a primary-scrollback live surface so the viewer stays live while terminal scrollback remains available. |
    | Scrollback | Launch mode prioritizes fullscreen live redraw; operators rely on pause or snapshot to inspect historical output. | Preserve launched fullscreen redraw and the existing local inspect affordances around pause, compact, and snapshot. | Attach mode keeps live output on primary scrollback, while still allowing local inspect controls without requiring `p` before scrolling. |
    | Mutability | The launched dashboard stays wired to runtime-owned refresh/subscription behavior and the broader host control flow around `orchestrator/src/cli/control/uiDataController.ts`. | Keep any mutating or host-owning behavior scoped to the launched runtime path. | Attach mode is read-only: it only reads authenticated `/ui/data.json` payloads and keeps inspect/snapshot actions local to the viewer surface instead of invoking host-mutating controls. |
- Dependencies / Integrations:
  - `bin/codex-orchestrator.ts`
  - `orchestrator/src/cli/control/controlStatusDashboard.ts`
  - `orchestrator/src/cli/control/uiDataController.ts`
  - `orchestrator/src/cli/delegationServer.ts` or a narrower shared control-endpoint loader
  - `README.md`
