## Summary
- Problem Statement: On March 30, 2026, the local `co-control-host` process stopped admitting newly eligible `Ready` issues even though the live Linear dispatch query still showed them as executable. The stale live instance stopped advancing `.runs/local-mcp/cli/control-host/provider-intake-state.json`, kept stale `CO-38` run truth, timed out both observability reads and refresh requests, and did not pick up `CO-39` / `CO-40` until the process was manually restarted.
- Desired Outcome: Make this class of provider refresh stall diagnosable and recoverable without restart as the first operator action by exposing explicit stuck/degraded evidence, recovering safely when possible, and covering the stale-live-instance / healthy-replay failure class with a focused reproducer or regression.
