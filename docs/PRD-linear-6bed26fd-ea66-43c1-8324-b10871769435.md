# PRD - CO: Exclude historical non-provider in_progress manifests from live CO STATUS current activity/runtime

## Added by Bootstrap 2026-04-03

## Traceability
- Linear issue: `CO-73` / `6bed26fd-ea66-43c1-8324-b10871769435`
- Linear URL: https://linear.app/asabeko/issue/CO-73/co-exclude-historical-non-provider-in-progress-manifests-from-live-co
- Follow-up context: `CO-67` / `f3193657-a549-43a2-8cff-50c5284df986`

## Summary
- Problem Statement: `CO STATUS` and `/api/v1/state` still overstate live work after `CO-67`. The local control-host intake snapshot shows no active provider claims and `latest_reason: provider_issue_released:not_active`, but the live host API still reports a large `running` set and inflated aggregate runtime because historical non-provider manifests with raw `in_progress` status are still admitted into the current running source set.
- Desired Outcome: current running rows and aggregate runtime come only from authoritative current activity. When provider intake has no active current claims and there is no other explicitly authoritative current activity, live `CO STATUS` and `/api/v1/state` should show `running = 0`, with regression coverage proving historical non-provider manifests no longer survive the current filter by falling back through generic task or run identifiers.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): tighten the current-activity contract for `CO STATUS` so it stops presenting historical generic manifests as live running work. The fix should be narrow, evidence-backed, and validated against the already-running control-host on this device with terminal and attach-viewer screenshots embedded directly in Linear.
- Success criteria / acceptance:
  - when provider intake has no active current claims and no other explicitly authoritative current activity, live `CO STATUS` and `/api/v1/state` show `running = 0`
  - historical non-provider `.runs/**/cli/*` manifests do not appear as current running activity unless they are explicitly current by contract
  - aggregate runtime is computed from the same authoritative current source set as the running table
  - a regression test covers null-provider historical manifests with no active current claim
  - final validation includes live host/API verification plus terminal proof on this device
  - real screenshots captured on this device are embedded directly in Linear for terminal proof and attach-viewer proof
- Constraints / non-goals:
  - preserve the existing `CO STATUS` / `/api/v1/state` surfaces instead of introducing a new viewer command
  - do not redesign `/ui` or rework pause/compact/snapshot UX beyond what truthful current activity requires
  - do not satisfy proof with rendered cards, JSON dumps, synthetic frames, or text-only summaries
