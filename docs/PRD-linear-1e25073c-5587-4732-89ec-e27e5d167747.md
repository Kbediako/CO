## Problem
`review/telemetry.json` can report `review_verdict: unknown` even when Codex review completed with `review_outcome: clean-success`, `finding_count: 0`, and clean natural-language review prose such as `No actionable defects were found...` or `No concrete correctness regressions were found...`.

That creates unnecessary provider-worker review waivers. The parser should recognize common clean review wording while still failing closed when findings are present or the text is genuinely ambiguous.

## Goal
Update semantic review verdict parsing so common clean review wording maps to `review_verdict: clean` without weakening findings detection, true-unknown handling, or provider-worker review handoff gates.

## Acceptance Criteria
- [ ] Semantic review parser recognizes `No actionable defects were found...` as `review_verdict: clean`.
- [ ] Semantic review parser recognizes `No concrete correctness regressions were found...` as `review_verdict: clean`.
- [ ] Existing `findings` detection still wins when P-prioritized or structured findings are present.
- [ ] Focused parser tests cover clean wording and negative/finding cases.
- [ ] Provider-worker review handoff behavior remains fail-closed for true `unknown` verdicts.

## Intent Checksum
Protected terms: `review/telemetry.json`, `review_verdict`, `clean`, `unknown`, `findings`, `finding_count`, `highest_finding_priority`, `review_outcome: clean-success`, semantic review parser, provider-worker review handoff gate.

Reject interpretations that treat every `unknown` verdict as clean, weaken `findings` handling, bypass provider-worker review gates, change PR/Linear lifecycle authority, or rewrite CO-492 advisory goal evidence behavior.

## Non-Goals
- No changes to CO-492 goal evidence capture.
- No provider-worker lifecycle gate weakening.
- No treating `review_outcome: clean-success` as sufficient when parsed findings exist.
- No suppressing P0/P1/P2/P3 finding lines.
- No broad review-wrapper rewrite outside semantic clean wording recognition.
- No changes to Linear, PR, ready-review, merge, or provider-worker lifecycle authority.

## Not Done If
- `review_verdict: unknown` is generally treated as clean or allowed to fail open.
- P-prioritized findings or structured findings are masked by clean wording.
- Tests do not cover the CO-492 observed clean phrases.
- Provider-worker handoff guidance no longer requires semantic clean or an explicit waiver.
- The change broadens into Linear, PR, ready-review, merge, or provider-worker lifecycle authority.
