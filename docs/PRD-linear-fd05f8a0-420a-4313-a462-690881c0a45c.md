- Problem Statement: CO-382 made fallback-expiry and large-refactor decisions explicit, but enforcement is still mostly checklist- and review-discipline-based. Agents can touch fallback, legacy, cached, break-glass, or minor-seam behavior without a parseable decision that repo guards can enforce.

## Large-Refactor Check
- `large refactor`: required when a new seam touches governed surfaces or fallback, legacy, cached, break-glass, or minor-seam patterns and the implementation cannot stay within a single bounded decision.
- `minor seam`: acceptable only when the packet records the exact fallback decision, the owner reference is current, and the guard can prove the new seam does not normalize routine fallback reliance.
- Parent implementation must enforce the large-refactor threshold through repo guards rather than relying on reviewer memory.
