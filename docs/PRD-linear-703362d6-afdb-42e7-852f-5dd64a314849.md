# PRD - CO: prepare and ship the next release after v0.1.38

## Summary
- Problem Statement: CO last shipped `v0.1.38` on `2026-03-03T07:07:00Z`, but current `origin/main` still carries `package.json` version `0.1.38` while the repo has moved through `124` merged PRs / `1238` non-merge commits, the public Codex posture has already advanced to `0.123.0`, and the unreleased delta now spans major operator, provider, runtime, review, and distribution work. CO-316 is the release-prep and ship lane that must turn that unreleased mainline delta into one truthful release instead of another narrow patch-style cut.
- Desired Outcome: produce a release-ready branch and release packet that justify the next semver bump, summarize the shipped themes since `v0.1.38`, pass the full release validation floor, merge cleanly, and then publish the signed Git tag, GitHub Release, and npm package from a clean `main` checkout.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): treat CO-316 as the actual release lane after the cloud/posture blockers were cleared by `CO-335`, not as another posture audit. The lane must release all main-branch work since `2026-03-03`, use truthful release notes, include downstream install and skill-refresh guidance when needed, and refuse to ship if the release still looks like a tiny patch or if release execution happens from a dirty non-`main` checkout.
- Success criteria / acceptance:
  - the next version bump is explicitly justified against the `v0.1.38..origin/main` delta
  - release notes cover the main shipped operator-facing and downstream-facing themes
  - release-prep work lands through the normal branch / PR / merge workflow
  - release execution happens from a clean `main` checkout after merge
  - full release validation floor passes before tagging
  - signed tag, GitHub release, and npm publish succeed
  - downstream install and bundled-skill refresh guidance is included when relevant
- Constraints / non-goals: no blind `0.1.39` patch bump, no release directly from this issue workspace, no skipping review/validation because the diff is large, no reopening Codex posture work already decided by `CO-335`, and no broad unrelated cleanup unless it blocks the release and is tracked separately.

## Not Done If
- The next tag is still framed or executed like a tiny patch despite the current delta size.
- Release notes omit the major work themes shipped since `v0.1.38`.
- Release prep or publish runs from a dirty non-`main` checkout.
- Required validation, review, or pack-smoke evidence is missing.
- Signed tag or publish prerequisites are not confirmed before execution.
- GitHub release or npm publish fails and the lane still moves to done.
