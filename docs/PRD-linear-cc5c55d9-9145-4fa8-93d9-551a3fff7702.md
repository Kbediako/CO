## Summary
- Problem Statement: linked-worktree helper flows are mutating the shared repo git config file in the main checkout (`/Users/kbediako/Code/CO/.git/config`) with a repo-local `[user]` override because they run plain `git config user.*` inside a linked worktree. The issue evidence already ties that pattern to the CO-102 archive-sync step, and the checked-in archive automation workflow currently models the same unsafe command shape for alternate identities.
- Desired Outcome: temporary linked-worktree flows inherit the operator's global git identity by default, including clearing any stale worktree-local identity override that would otherwise mask inherited identity after a prior explicit helper run, any explicit alternate identity stays isolated to the linked worktree only, and the repo owns a regression-proof verification path showing the shared repo git config file gains no new `[user]` override from the helper flow. If Git requires `extensions.worktreeConfig=true` before worktree-local config can be used, that one-time non-user shared-config precondition must be explicit and limited.
