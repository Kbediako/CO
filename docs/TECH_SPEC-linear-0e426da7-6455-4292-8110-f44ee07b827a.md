## Proposed Design
- Add or reuse one shared lineage contract that orders checks as:
  1. require launch provenance where the current seam expects it
  2. require matching current lineage fields such as `parent_run_id` and issue identity
  3. allow `older canonical parent manifest` only as a bounded backstop for stale path/layout drift
  4. allow issue-workspace rebinding to rewrite artifact paths only after the current lineage is established
  5. make source-0 and projection surfaces consume the same current-lineage decision
- Keep the result narrow: provenance and lineage correctness only, not generic workspace-root or artifact-root redesign.
