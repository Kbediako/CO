## Cloud Canary Boundary
- `cloud_fallback` is an execution-mode fallback contract, not the same object as `runtime_fallback`.
- Required cloud execution still needs `CODEX_CLOUD_ENV_ID=<env-id> CODEX_CLOUD_CANARY_REQUIRED=1 npm run ci:cloud-canary` or an exact blocker/waiver in the owning lane.
- The fallback cloud canary, `CODEX_CLOUD_ENV_ID="" CODEX_CLOUD_CANARY_REQUIRED=1 CLOUD_CANARY_EXPECT_FALLBACK=1 npm run ci:cloud-canary`, only proves the expected local MCP fallback contract. It must not satisfy required cloud execution.
- CO-396 implementation may update cloud-mode docs to clarify this boundary, but must not relax cloud canary gates.
