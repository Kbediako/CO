## Summary
- Problem: `co-status attach` already uses `primary scrollback`, but each refresh still appends a full new `CO STATUS` frame. Operators scrolling during live monitoring therefore see stacked full-screen history instead of one pinned live surface plus honest history.
- Outcome: keep attach read-only and live on `primary scrollback`, switch it to bounded in-place updates, and tighten the same protected live surfaces called out in the issue: countdowns, runtime/age, rate limits, retry text, and event text.

## Contract
- User request translation: preserve live scrollable `CO STATUS`, stop accumulated duplicate full-frame history in `co-status attach`, and make the visible live status text readable and trustworthy enough for active operator debugging.
- Acceptance:
  - `co-status attach` stays read-only for an already-running control-host.
  - live attach no longer accumulates duplicate full-frame `CO STATUS` history in primary scrollback.
  - attach uses a pinned or in-place primary update strategy rather than appending a full new frame each refresh.
  - launch-mode `co-status` alternate-screen behavior stays distinct and unregressed.
  - `Next refresh`, `AGE / TURN`, retry countdowns, and reset windows use higher-order readable durations.
  - Codex / Linear rate-limit output uses the requested percentage and spacing semantics.
  - retry and `EVENT` text are meaningfully better than generic fallback wording.
  - final validation includes real-device screenshots embedded directly in Linear after visual inspection.
- Protected terms: `CO STATUS`, `co-status attach`, `primary scrollback`, `alternate screen`, `controlStatusDashboard.ts`, `coStatusAttachCliShell.ts`, `ControlStatusDashboard.test.ts`, `CoStatusAttachCliShell.test.ts`, `CO-74`, `CO-67`, `CO-83`.
- Reject:
  - reverting attach to `alternate screen` and calling that done
  - treating this as only the old `CO-67` paused duplication bug
  - accepting stacked full-screen history because attach intentionally runs on primary
  - fixing only docs/help text without changing live rendering behavior
  - reopening the broader telemetry lane unless validation proves the remaining defect left the attach/view seam
- Non-goals:
  - browser `/ui` redesign
  - pause-first fallback as the intended operator workflow
  - broad styling churn outside the protected readability/truth surfaces

## Parity Matrix
- Current:
  - attach is live on primary but still redraws whole frames into scrollback
  - long countdowns, age/runtime, rate limits, retry rows, and event text still carry the readability defects called out in the issue
- Reference:
  - operators should see one live `CO STATUS` surface plus honest history
  - long-lived live status text should be readable without manual time/limit decoding
- Target:
  - attach updates one pinned live block instead of stacking full-frame copies
  - countdown, age/runtime, rate-limit, retry, and event surfaces are readable and truthful for incident use

- the shipped fix simply moves attach back to alternate screen
