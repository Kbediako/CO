## Summary
- Problem Statement: GitHub Actions warns that Node.js 20 action runtimes are deprecated and will be forced to Node 24 by default starting `2026-06-02`. CO workflows still use action majors whose `action.yml` declares `node20`, including the issue-named checkout, setup-node, and create-pull-request actions plus additional audited workflow actions.
- Desired Outcome: every workflow under `.github/workflows/**` uses Node 24-compatible action majors where supported, while preserving workflow behavior, permissions, PR/archive automation behavior, release behavior, cache behavior, and the repo's explicit installed Node versions.

## User Request Translation (Context Anchor)
- User intent / needs: complete CO-375 by auditing all GitHub Actions workflow `uses:` references and upgrading Node 20-backed action majors before GitHub's `2026-06-02` runner switch changes behavior by default.
- Success criteria / acceptance:
  - audit all nine workflow files under `.github/workflows/**`
  - upgrade supported Node 20 action majors to Node 24-compatible majors
  - keep `node-version` values, permissions, cache inputs, artifact names/paths, release publish behavior, and archive PR behavior unchanged unless a target action requires an explicit compatibility adjustment
  - validate Core Lane, Cloud Canary, archive automation, release-relevant workflows, and repo guardrails after the change
- Constraints / non-goals:
  - do not change CO's Node package/runtime baseline in CI; `node-version: 20` and `node-version: 22` are separate from action runtime deprecation
  - do not redesign workflow schedules, permissions, release signing, npm publishing, archive branch handling, or auto-merge behavior
  - do not expand into Codex CLI version policy changes

## Not Done If
- Any `.github/workflows/**` `uses:` reference still points to a known Node 20-backed action major where a Node 24-compatible supported major is available.

- Core Lane, Cloud Canary, archive automation, or release-relevant workflow paths are not validated by scan plus repo gates or a precise blocker note.
