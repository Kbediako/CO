## Scope
- Keep the fix bounded to provider-worker closeout/reporting, deterministic invalid mutation suppression, and proof-sidecar truth.
- Preserve existing Linear validation contracts and fail closed when artifacts disagree.
- Avoid schema migrations, workpad-model rewrites, and historical backfill.

## Design
- Add a shared helper in `orchestrator/src/cli/control/providerLinearWorkerTruth.ts` for terminal proof status, summary wording, deterministic mutation suppression, and auxiliary proof-harness detection.
- Update `orchestrator/src/cli/services/commandRunner.ts` so provider-worker closeout uses authoritative proof plus fresh review telemetry, never success-shaped `summaryHint` text on failures, and ignores stale review telemetry from earlier attempts in the same run.
- Update `orchestrator/src/cli/control/selectedRunProjection.ts` so stale optimistic manifests can be overridden by newer failed proof and auxiliary `manual-live-proof` runs are excluded from active truth discovery.
- Update `orchestrator/src/cli/providerLinearWorkerRunner.ts` to carry deterministic invalid mutation suppressions into subsequent prompts for the same attempt.
- Update manifest/pipeline cleanup in `orchestrator/src/cli/run/manifest.ts` and `orchestrator/src/cli/services/orchestratorLocalPipelineExecutor.ts` so stale failure summary lines are removed after later success.
