## Summary
- Objective: make provider-worker closeout/reporting authoritative so the terminal result and review telemetry agree across manifest-backed closeout surfaces, while suppressing deterministic invalid provider mutations and removing proof-sidecar noise from operator truth.
- Scope:
  - authoritative terminal closeout derivation for manifest, run summary, out/state, and workpad surfaces
  - deterministic invalid provider mutation classification and same-attempt suppression
  - proof-sidecar finalization or exclusion from active provider-worker truth
  - focused regression coverage for the `CO-78` failure class
- Constraints:
  - no historical artifact backfill
  - no workpad model rewrite
  - no unrelated provider-worker reporting polish

## Technical Requirements
- Functional requirements:
  - primary closeout surfaces must derive from the authoritative terminal lane result instead of partial milestone text
  - review closeout language must reflect authoritative review telemetry, including `clean-success`, `bounded-success`, `failed-boundary`, and `failed-other`
  - deterministic invalid provider mutations must be classified non-retryable within the same attempt
  - auxiliary proof harness runs must no longer appear as active provider-worker work when they are merely helper artifacts
  - focused regression coverage must reproduce mixed partial-progress / terminal-failure closeout behavior
- Non-functional requirements (performance, reliability, security):
  - prefer fail-closed truth when artifacts disagree
  - preserve current validation rules for follow-up creation instead of weakening mutation contracts
  - keep runtime/status truth surfaces machine-checkable and auditable
- Interfaces / contracts:
  - provider-worker proof and attempt state: `orchestrator/src/cli/providerLinearWorkerRunner.ts`
  - terminal closeout/report summary derivation: `orchestrator/src/cli/services/commandRunner.ts`
  - operator-facing truth projection: `orchestrator/src/cli/control/selectedRunProjection.ts`
  - shared provider-worker truth helpers: `orchestrator/src/cli/control/providerLinearWorkerTruth.ts`

## Architecture & Data
- Architecture / design adjustments:
  - centralize closeout status wording around one authoritative terminal result plus one authoritative review telemetry snapshot
  - let `commandRunner` fail closed on authoritative provider-worker proof or failed review telemetry even when the subprocess exit path looks superficially successful
  - carry deterministic invalid mutation classification through the attempt so retries are suppressed in subsequent prompts
  - exclude proof helper artifacts from active runtime aggregation
- Data model changes / migrations:
  - no persistent schema migration expected
  - proof or closeout payloads may need additional explicit classification fields if the current status vocabulary is too lossy
- External dependencies / integrations:
  - Linear follow-up creation helper contract
  - review telemetry payloads written by the standalone review wrapper
  - operator-facing runtime aggregation that currently reads provider-worker proof state

## Parity / Alignment Matrix
- Current truth:
  - primary closeout surfaces can disagree after a lane partially succeeds before terminal failure
  - deterministic invalid follow-up creation can be retried as if transient
  - proof sidecars can linger as `in_progress` and contaminate active runtime truth
- Reference truth:
  - authoritative terminal result plus authoritative review telemetry should shape all closeout surfaces
  - deterministic invalid mutations are fail-closed within an attempt
  - helper proof runs should not masquerade as operator work
- Target truth / intended delta:
  - one lane result, one review classification, and one operator-facing truth across closeout surfaces
  - deterministic invalid retries suppressed after first classification
  - proof helper sidecars no longer count as live provider-worker activity
- Explicitly out-of-scope differences:
  - rewriting already-emitted historical artifacts
  - broad provider-worker UX copy changes
