## Risks & Mitigations
- Risk: implementation fetches or mutates repo state while checking posture.
  - Mitigation: make non-destructive behavior a hard requirement and test no-remote/no-fetch behavior through fixtures or mocked Git calls.
- Risk: warning text duplicates stale posture policy.
  - Mitigation: parent should centralize the current-main posture summary or keep it in a tested helper tied to `docs/guides/codex-version-policy.md`.
- Risk: dirty local edits are mistaken for stale branch posture drift.
  - Mitigation: separate state fields and separate human messages are required.
- Risk: downstream checkouts without `origin/main` see confusing failures.
  - Mitigation: no-remote classification is required and must fail soft.
