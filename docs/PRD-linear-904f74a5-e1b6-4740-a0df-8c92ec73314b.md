## User Request Translation

Rebaseline CO's public marketplace guidance and downstream smoke coverage to the live Codex CLI `0.122.0` command surface without changing the underlying packaged marketplace architecture or the supported npm install path. The worker must update the public docs, launcher guidance, pack-smoke prerequisite logic, pack-smoke invocation, and related tests so downstream users are told to run `codex plugin marketplace add` and can remove the marketplace with `codex plugin marketplace remove`.

This is explicitly a follow-up to `CO-196`, not a new packaging architecture lane and not a general Codex `0.122.0` posture promotion decision. The implementation should stay narrow to the command-surface rebaseline plus the smoke/test surfaces needed to keep that contract enforced.

## Problem Statement

`codex-cli 0.122.0` changed the user-facing marketplace command surface. Local evidence in this workspace shows:

- `codex --version` reports `codex-cli 0.122.0`
- `codex marketplace add --help` fails with `error: unrecognized subcommand 'add'`
- `codex plugin marketplace add --help` succeeds
- `codex plugin marketplace remove --help` succeeds

Official upstream release evidence also moved forward: GitHub release `rust-v0.122.0` is marked `Latest` and published on `2026-04-20`.

CO still hard-codes the old `codex marketplace add` wording in the current public install docs, launcher recovery text, pack-smoke prerequisite detection, pack-smoke invocation, related tests, and workflow pins that provide the smoke environment. That makes the current downstream guidance misleading on the latest stable CLI.

- Pack-smoke detects and invokes the `codex plugin marketplace add` command surface truthfully for `0.122.0`.

- npm remains the supported baseline install path and the shipped marketplace/plugin packaging layout remains unchanged.
