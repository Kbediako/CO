# PRD: CO-268 Marketplace Docs And Smoke Coverage Rebaseline To Codex 0.122 Plugin Marketplace Commands

## Summary
- Problem Statement: Codex CLI `0.122.0` moved the user-facing marketplace add/remove command surface under `codex plugin marketplace ...`, while CO still documents and tests the older `codex marketplace add` path in public setup, launcher recovery, pack-smoke prerequisites, and workflow smoke coverage.
- Desired Outcome: CO downstream setup, launcher guidance, pack-smoke logic, tests, and smoke workflow pins truthfully match Codex CLI `0.122.0` without changing the npm baseline path or the CO-196 packaged marketplace architecture.

## User Request Translation (Context Anchor)
- User intent / needs (in your own words): Complete Linear `CO-268` as a narrow command-surface rebaseline from `codex marketplace ...` to `codex plugin marketplace ...` for current stable Codex CLI `0.122.0`.
- Success criteria / acceptance: Capture official release and local command-surface evidence, update protected docs/code/test/workflow surfaces, preserve npm and packaging boundaries, and link back to `CO-196` plus posture lineage.
- Constraints / non-goals: Do not treat this as a new packaging feature, a general `0.122.0` posture promotion, a provider-worker authority change, app-server routing work, or npm path de-support.

## Intent Checksum
- Exact user wording / phrases to preserve: `Codex CLI 0.122.0`, `codex plugin marketplace add`, `codex plugin marketplace remove`, `codex marketplace add`, `README.md`, `docs/public/downstream-setup.md`, `scripts/pack-smoke.mjs`, `tests/pack-smoke.spec.ts`, `plugins/codex-orchestrator/launcher.mjs`, `CO-196`.
- Protected terms / exact artifact and surface names: `rust-v0.122.0`, `@openai/codex@0.122.0`, `codex-orchestrator:canonical-owner-key=codex-0122-plugin-marketplace-command-rebaseline`, `.github/workflows/core-lane.yml`, `.github/workflows/pack-smoke-backstop.yml`, `.github/workflows/release.yml`, `docs/skills-release.md`.
- Nearby wrong interpretations to reject: new marketplace/plugin packaging architecture, removal of npm baseline install guidance, broad Codex `0.122.0` posture promotion, provider-worker authority changes, app-server routing changes, or plugin architecture redesign.
