## Summary
- Problem Statement: appserver child-lane stale invalidation can only be safe when the parent can prove the exact `providerLinearChildLaneRunner` process being evaluated. A `heartbeat stale` signal and `no proof output` are useful symptoms, but they are not enough if `provider_linear_child_lane_runner_pid`, `provider_linear_child_lane_runner_started_at`, and `runner_alive` are missing, ambiguous, or vulnerable to PID reuse.
- Desired Outcome: emit `stale_invalidation_candidate` only when `runtime_mode=appserver`, the original runner identity is known, the same runner is proven not alive, the heartbeat is stale, and no proof output exists. Ambiguous or unknown process identity remains fail-closed and must not produce a stale invalidation candidate.
